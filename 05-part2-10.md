# 10 リストの続き (More on Lists)

Lists are a versatile form of data that come with almost all languages now.

リストは、現在ほとんどすべての言語に備わっている汎用性の高いデータ形式です。

Programmers have used them to build large applications, artificial intelligences, distributed systems, and more.

プログラマはリストを使って、大規模なアプリケーション、人工知能、分散システムなどを構築してきました。

This chapter illustrates some ideas from this world, including functions that create lists, data representations that call for structures inside of lists, and representing text files as lists.

この章では、この世界のいくつかの考え方を説明します。リストを生成する関数、リストの中に構造体を必要とするデータ表現、そしてテキストファイルをリストとして表現することなどが含まれます。

## 10.1 リストを生成する関数 (Functions that Produce Lists)

Here is a function for determining the wage of an hourly employee:

以下は、時給制の従業員の賃金を計算する関数です。

```racket
; Number -> Number
; computes the wage for h hours of work
(define (wage h)
  (* 12 h))
```

It consumes the number of hours worked and produces the wage.

この関数は労働時間を受け取り、賃金を生成します。

A company that wishes to use payroll software isn't interested in this function, however.

しかし、給与計算ソフトウェアを使いたい会社は、この関数には興味がありません。

It wants one that computes the wages for all its employees.

会社は、すべての従業員の賃金を計算する関数を必要としています。

Call this new function wage*.

この新しい関数を wage* と呼びましょう。

Its task is to process all employee work hours and to determine the wages due to each of them.

この関数の仕事は、すべての従業員の労働時間を処理し、それぞれに支払うべき賃金を決定することです。

For simplicity, let's assume that the input is a list of numbers, each representing the number of hours that one employee worked, and that the expected result is a list of the weekly wages earned, also represented with a list of numbers.

簡単のため、入力は数値のリストで、各数値は1人の従業員が働いた時間数を表し、期待される結果も数値のリストで、各数値はその週に稼いだ賃金を表すと仮定しましょう。

Since we already have a data definition for the inputs and outputs, we can immediately move to the second design step:

入力と出力に対するデータ定義はすでに持っているので、すぐに設計の第2ステップに進むことができます。

```racket
; List-of-numbers -> List-of-numbers
; computes the weekly wages for the weekly hours
(define (wage* whrs)
  '())
```

Next you need some examples of inputs and the corresponding outputs.

次に、入力の例とそれに対応する出力の例が必要です。

So you make up some short lists of numbers that represent weekly hours:

そこで、週の労働時間を表す短い数値リストをいくつか作ってみましょう。

```racket
'()                ->  '()
(cons 28 '())      ->  (cons 336 '())
(cons 4 (cons 2 '())) -> (cons 48 (cons 24 '()))
```

In order to compute the output, you determine the weekly wage for each number on the given input list.

出力を計算するために、与えられた入力リストの各数値に対して週の賃金を計算します。

For the first example, there are no numbers on the input list so the output is '().

最初の例では、入力リストに数値がないので、出力は '() になります。

Make sure you understand why the second and third expected outputs are what you want.

2番目と3番目の期待される出力がなぜそうなるのか、理解するようにしてください。

Given that wage* consumes the same kind of data as several other functions from Lists and given that a template depends only on the shape of the data definition, you can reuse this template:

wage* が「リスト」の章の他のいくつかの関数と同じ種類のデータを消費し、テンプレートがデータ定義の形にのみ依存することを考えると、このテンプレートを再利用できます。

```racket
(define (wage* whrs)
  (cond
    [(empty? whrs) ...]
    [else (... (first whrs) ...
               ... (wage* (rest whrs)) ...)]))
```

In case you want to practice the development of templates, use the questions from figure 52.

テンプレートの作成を練習したい場合は、図52の質問を使ってください。

It is now time for the most creative design step.

これで最も創造的な設計ステップの時間です。

Following the design recipe, we consider each cond line of the template in isolation.

設計レシピに従って、テンプレートの各 cond 行を個別に考えます。

For the non-recursive case, (empty? whrs) is true, meaning the input is '().

再帰しないケースでは、(empty? whrs) が真であり、入力は '() であることを意味します。

The examples from above specify the desired answer, '(), and so we are done.

上記の例から、望ましい答えは '() であると指定されているので、ここは完了です。

In the second case, the design questions tell us to state what each expression of the template computes:

2番目のケースでは、設計の質問が、テンプレートの各式が何を計算するかを述べるよう指示しています。

- (first whrs) yields the first number on whrs, which is the first number of hours worked.

- (first whrs) は whrs の最初の数値を返します。これは最初の労働時間数です。

- (wage* (rest whrs)) computes the list of wages for all but the first item on whrs.

- (wage* (rest whrs)) は、whrs の最初の項目を除くすべての項目に対する賃金のリストを計算します。

The key is to rely on these facts when you formulate the expression that computes the result in this case, even if the function is not yet defined.

重要なのは、関数がまだ定義されていなくても、このケースで結果を計算する式を定式化するときに、これらの事実を頼りにすることです。

Since we already have the list of wages for all but the first item of whrs, the function must perform two computations to produce the expected output for the entire whrs: compute the weekly wage for (first whrs) and construct the list that represents all weekly wages for whrs.

whrs の最初の項目を除くすべての賃金のリストはすでに持っているので、関数は whrs 全体に対する期待される出力を生成するために2つの計算を行う必要があります。 (first whrs) の週賃金を計算し、whrs のすべての週賃金を表すリストを構築することです。

For the first part, we reuse wage.

最初の部分では、wage を再利用します。

For the second, we cons the two pieces of information together into one list:

2番目の部分では、2つの情報を cons で1つのリストにまとめます。

```racket
(define (wage* whrs)
  (cond
    [(empty? whrs) '()]
    [else (cons (wage (first whrs))
                (wage* (rest whrs)))]))
```

And with that, the definition is complete.

これで定義は完成です。

Generalize convert-euro to the function convert-euro*, which consumes an exchange rate and a list of US$ amounts and converts the latter into a list of € amounts.

convert-euro を convert-euro* に一般化せよ。この関数は為替レートとUSドルの金額のリストを受け取り、それをユーロの金額のリストに変換する。

Generalize subst-robot to substitute.

subst-robot を substitute に一般化せよ。

The latter consumes two strings, called new and old, and a list of strings.

後者は new と old と呼ばれる2つの文字列と、文字列のリストを受け取る。

It produces a new list of strings by substituting all occurrences of old with new.

old のすべての出現を new に置き換えた新しい文字列のリストを生成する。

## 10.2 構造体を含むリスト (Structures in Lists)

Lists are not just for atomic data such as numbers and strings.

リストは数値や文字列のような原子データのためだけにあるわけではありません。

Many applications require lists of structures.

多くのアプリケーションでは、構造体のリストが必要になります。

Here is a data definition for lists of Posns:

以下は Posn のリストに対するデータ定義です。

```racket
; A List-of-posns is one of:
; - '()
; - (cons Posn List-of-posns)
```

The template for functions that consume such lists is the usual one:

このようなリストを消費する関数のテンプレートは、いつものものです。

```racket
(define (fun-for-lop lop)
  (cond
    [(empty? lop) ...]
    [else (... (first lop) ...
               ... (fun-for-lop (rest lop)) ...)]))
```

When you use (first lop) in the else clause, you know that it is a Posn.

else 節で (first lop) を使うとき、それが Posn であることがわかっています。

Therefore you may apply Posn selectors to it.

したがって、その Posn に対してセレクタを適用することができます。

## 10.3 リストのリスト、ファイル (Lists in Lists, Files)

Lists can contain lists.

リストはリストを含むことができます。

For example, here is a data definition for lists of lists of numbers:

例えば、以下は数値のリストのリストに対するデータ定義です。

```racket
; A List-of-list-of-numbers is one of:
; - '()
; - (cons List-of-numbers List-of-list-of-numbers)
```

Functions that process such nested lists follow the usual template, but the (first ...) expression itself is a list, so you apply another list-processing function to it.

このような入れ子になったリストを処理する関数は、通常のテンプレートに従いますが、(first ...) の式自体がリストであるため、それに対して別のリスト処理関数を適用します。

You can also represent files as lists.

ファイルをリストとして表現することもできます。

Using the batch-io teachpack, you can read the content of a file as a list of lines or a list of words.

batch-io ティーチパックを使うと、ファイルの内容を行のリストや単語のリストとして読み込むことができます。

## 10.4 グラフィカルエディタの再訪 (A Graphical Editor, Revisited)

In chapter 5 we designed a simple graphical editor.

第5章でシンプルなグラフィカルエディタを設計しました。

With lists we can represent the text before and after the cursor as lists of 1Strings.

リストを使うことで、カーソルの前後のテキストを1Stringsのリストとして表現できます。

```racket
(define-struct editor [pre post])
; An Editor is a structure:
;   (make-editor List-of-1Strings List-of-1Strings)
```

This representation makes it easy to insert and delete characters at the cursor position.

この表現により、カーソル位置での文字の挿入と削除が容易になります。
