# 8 リスト (Lists)

これまで自己参照的な定義に出会ったことはおそらくないでしょう。国語（英語）の教師は間違いなくこのような定義を避けますし、多くの数学の授業でもこうした定義は曖昧に扱われます。しかし、プログラマは曖昧さを許容できません。プログラマの仕事には正確さが必要です。一般に、定義には自分自身への参照が複数含まれることがありますが、この章ではリストを皮切りに、自己参照をちょうど1つだけ必要とする有用な例を紹介します。

リストを導入することで、私たちが学ぶことのできるアプリケーション의幅も広がります。この章では例を通じて直感を丁寧に築き上げるとともに、自己参照データを体系的に処理する関数を作成する方法を説明する次の章での設計レシピの改訂へと動機づけます。

## 8.1 リストの作成 (Creating Lists)

私たちは皆、日常的にリストを作成しています。食料品の買い物に行く前には、購入したい品目のリストを書き留めます。毎朝ToDoリストを書く人もいます。12月には、多くの子供たちがクリスマスの欲しいものリストを用意します。パーティーを計画するときは、招待者のリストを作ります。情報をリストの形で整理することは、私たちの生活のいたるところにあります。

情報がリストの形で与えられるなら、そのようなリストをBSL（Beginning Student Language）のデータとして表現する方法を明確に学ぶ必要があります。実際、リストは非常に重要であるため、BSLにはデカルト座標の点（`posn`）と同様に、リストの作成 and 操作のための組み込みサポートが備わっています。点とは対照的に、リスト of データ定義は常に自分で書く必要があります。しかし、まずは順を追って進めましょう。リストの作成から始めます。

リストを形成するとき、私たちは常に空のリスト（empty list）から始めます。BSLでは、空のリストを以下のように表現します。

```racket
'()
```

これは「エンプティ（empty）」と発音され、「空リスト（empty list）」の略です。`#true` や `5` と同様に、`'()` は単なる定数です。リストに何かを追加するときは、別のリストを構築します。BSLでは、`cons` 操作がこの目的を果たします。例えば、

```racket
(cons "Mercury" '())
```

は、`'()` リストと文字列 `"Mercury"` からリストを構築します。図44は、このリストを構造体と同じように視覚的に表現したものです。`cons` の箱には、`first` と `rest` という2つのフィールドがあります。この特定の例では、`first` フィールドに `"Mercury"` が入り、`rest` フィールドに `'()` が入っています。

> **Figure 44: リストの構築**
> 
> *   `list`: `(cons "Mercury" '())`
>     *   `diagram`: `[first: "Mercury" | rest: '()]`
> *   `list`: `(cons "Venus" (cons "Mercury" '()))`
>     *   `diagram`: `[first: "Venus" | rest: ●] ---> [first: "Mercury" | rest: '()]`
> *   `list`: `(cons "Earth" (cons "Venus" (cons "Mercury" '())))`
>     *   `diagram`: `[first: "Earth" | rest: ●] ---> [first: "Venus" | rest: ●] ---> [first: "Mercury" | rest: '()]`

1つの要素を持つリストができたら、再び `cons` を使って2つの要素を持つリストを構築できます。以下はその一例です。

```racket
(cons "Venus" (cons "Mercury" '()))
```

また、別の例を示します。

```racket
(cons "Earth" (cons "Mercury" '()))
```

図44の中央の行は、2つの要素を含むリストをどのようにイメージできるかを示しています。これも2つのフィールドを持つ箱ですが、今回の `rest` フィールドには別の箱が入っています。実際、同じ図の上の行にある箱が入っています。

最後に、3つの要素を持つリストを構築します。

```racket
(cons "Earth" (cons "Venus" (cons "Mercury" '())))
```

図44の最後の行は、3つの要素を持つリストを表しています。その `rest` フィールドには、さらに箱を含む箱が入っています。このように、リストを作成する際には、箱の中に箱を入れ、その中にさらに箱を入れるということを繰り返します。一見すると奇妙に思えるかもしれませんが、これは誕生日に贈られる入れ子のおもちゃ（マトリョーシカ）や、重ねられたコップのようなものです。唯一の違いは、BSLのプログラムは、職人が物理的な箱を重ねるよりもはるかに深くリストを入れ子にできるという点です。

優れた画家であっても、深く入れ子になった構造を描くのは難しいため、コンピュータ科学者は代わりに「箱と矢印（box-and-arrow）」の図を使用します。図45は、図44の最後の行を再配置する方法を示しています。各 `cons` 構造体は別々の箱になります。もし `rest` フィールドが複雑すぎて箱の中に描けない場合は、代わりに黒丸（●）を描き、そこからそれが含む箱へと矢印付きの線を引きます。箱の配置方法によって、2種類の図が得られます。

図45の上の行に示されている最初の配置は、箱が作成された順に並べたものです。下の行に示されている2番目の配置は、それらが `cons` された順に並べたものです。したがって、2番目の図を見れば、リストの長さにかかわらず、そのリストに `first` を適用したときに何が生成されるかがすぐに分かります。このため、プログラマは2番目の配置を好みます。

```racket

```

このリストの最初の要素は文字列、2番目は数値、最後はブール値です。このリストは、従業員の氏名、社歴（年数）、会社を通じて健康保険に加入しているかどうかという3つのデータからなる人事レコードの表現と考えることができます。あるいは、ゲームの仮想プレイヤーを表していると考えることもできます。データ定義がなければ、そのデータが何についてのものであるかを知ることはできません。

以下は、`cons` を含む最初のデータ定義です。

```racket
;; A 3LON is a list of three numbers:
;;   (cons Number (cons Number (cons Number '())))
;; interpretation: a point in 3-dimensional space
```

もちろん、このデータ定義は、他者が構造体のインスタンスにコンストラクタを使用するのと同じように `cons` を使用しており、ある意味で `cons` は特別なコンストラクタにすぎません。このデータ定義が示せていないのは、任意の長さのリスト（要素が何もないリスト、1つのリスト、2つのリスト、10のリスト、あるいはおそらく 1,438,901 個の要素を持つリスト）をどのように形成するかということです。

そこで、もう一度試してみましょう。

```racket
;; A List-of-names is one of:
;; -- '()
;; -- (cons String List-of-names)
;; interpretation: a list of invitees, by last name
```

深呼吸をして、もう一度読んでください。このデータ定義は、これまでに遭遇した中で最も珍しい定義の1つです。なぜなら、自分自身を参照する定義を見たことがないからです。これが意味をなすかどうかも明らかではありません。結局のところ、国語の先生に「『机』とは『机』のことである」と説明しても、「デタラメだ！」と言われるだけでしょう。自己参照的な定義は単語の意味を説明しないからです。

しかし、コンピュータ科学やプログラミングにおいては、自己参照的な定義が中心的な役割を果たしており、注意を払えば実際に意味をなします。ここで「意味をなす」とは、データ定義を本来の目的、すなわち定義されているクラスに属するデータの例を生成したり、与えられたデータが定義されたクラスに属しているかどうかを確認したりするために使用できることを意味します。この観点から見ると、`List-of-names` の定義は完全に意味をなしています。

最低限、項目の最初の節（`'()`）を使って、1つの例として `'()` を生成できます。`'()` が `List-of-names` の要素であることが分かれば、2番目の例を作るのも簡単です。

```racket
(cons "Findler" '())
```

ここでは、文字列（`String`）と `List-of-names` の唯一の既知のリスト（`'()`）を使用して、項目化の2番目の節に従ってデータを生成しています。同じルールで、この種のリストをさらに多く生成できます。

```racket
(cons "Flatt" '())
(cons "Felleisen" '())
(cons "Krishnamurthi" '())
```

これらのリストにはそれぞれ1つの名前（文字列として表現されたもの）しか含まれていませんが、データ定義の2行目を使用して、より多くの名前を含むリストを作成することもできます。

```racket
(cons "Felleisen" (cons "Findler" '()))
```

このデータは `List-of-names` に属します。なぜなら、`"Felleisen"` は文字列であり、`(cons "Findler" '())` はすでに確認された `List-of-names` の要素だからです。

## 8.2 '() とは何か、cons とは何か (What Is '(), What Is cons)

少し立ち止まって、`'()` と `cons` を詳しく見てみましょう。前述したように、`'()` は単なる定数です。`5` や `"this is a string"` などの定数と比較すると、これは関数名や変数のように見えます。しかし、`#true` や `#false` と比較すれば、これが単にBSLにおける空のリストの表現であることが簡単にわかるはずです。

評価ルールの観点から見ると、`'()` は数値やブール値、文字列などとは異なる、新しい種類のアトミックな（原子的な）値です。また、`posn` のような複合値（構造体インスタンス）ではありません。実際、`'()` は非常にユニークであり、それ自体で独自の値を形成します。そのため、この値のクラスには、`'()` だけを認識し、他には何も認識しない述語が用意されています。

```racket
;; Any -> Boolean
;; 与えられた値が '() であるかを判定する
(define (empty? x) ...)
```

すべての述語と同様に、`empty?` はBSLのあらゆる値に適用できます。これが `#true` を生成するのは、`'()` に適用されたときだけです。

```racket
> (empty? '())
#true
> (empty? 5)
#false
> (empty? "hello world")
#false
> (empty? (cons 1 '()))
#false
> (empty? (make-posn 0 0))
#false
```

次に、`cons` について考えます。これまで見てきたことはすべて、`cons` が構造体型定義によって導入されるようなコンストラクタにすぎないことを示唆しています。より正確には、`cons` は2つのフィールドを持つ構造体のコンストラクタであるように見えます。最初のフィールドは任意の値、2番目のフィールドはリストに類する値が入ります。以下の定義は、このアイデアを BSL で表現したものです。

```racket
(define-struct pair [left right])
;; A ConsPair is a structure:
;;   (make-pair Any Any).

;; Any Any -> ConsPair
(define (our-cons a-value a-list)
  (make-pair a-value a-list))
```

唯一のキャッチ（注意点）は、`our-cons` が2番目の引数としてすべての可能な BSL 値を受け入れるのに対し、`cons` はそうではないという点です。以下の実験がこれを検証しています。

```racket
> (cons 1 2)
cons: second argument must be a list, but received 1 and 2
```

言い換えれば、`cons` は実際には「チェックされた関数」（第I部「項目化と構造体」で説明したもの）であり、以下のような洗練されたデータ定義を示唆しています。

```racket
;; A ConsOrEmpty is one of:
;; -- '()
;; -- (make-pair Any ConsOrEmpty)
;; interpretation: ConsOrEmpty is the class of all lists

;; Any Any -> ConsOrEmpty
(define (our-cons a-value a-list)
  (cond
    [(empty? a-list) (make-pair a-value a-list)]
    [(pair? a-list) (make-pair a-value a-list)]
    [else (error "cons: second argument ...")]))
```

もし `cons` がチェックされたコンストラクタ関数であるなら、結果の構造体からどのようにして要素を取り出すのか疑問に思われるかもしれません。結局のところ、第I部「構造体の追加」では、構造体を使ったプログラミングにはセレクタが必要であると説明しました。`cons` 構造体には2つのフィールドがあるため、2つのセレクタがあります。それは `first` と `rest` です。これらも、`pair` 構造体を使って簡単に定義できます。

```racket
;; ConsOrEmpty -> Any
;; 与えられたペアの left 部分を抽出する
(define (our-first a-list)
  (if (empty? a-list)
      (error 'our-first "...")
      (pair-left a-list)))
```

立ち止まって、`our-rest` を定義してみてください。

もしあなたのプログラムが `pair` の構造体型定義に直接アクセスできると、`right` フィールドに `'()` や別の `pair` を含まないような `pair` のインスタンスを簡単に作成できてしまいます。このような不適切なインスタンスが意図的または偶発的に作成されると、関数やプログラムが奇妙な方法で破損する原因となります。そのため、BSLではこれらの問題を回避するために、`cons` の実際の構造体型定義を隠蔽しています。第III部「局所定義」では、プログラムでこのような定義を隠蔽する方法の1つを示しますが、現時点ではその必要はありません。

> **Figure 46: リストのプリミティブ**
> 
> *   `'()` : 特別な値。主に空リストを表現するために使用される。
> *   `empty?` : `'()` だけを認識する述語。
> *   `cons` : 2つのフィールドを持つインスタンスを作成する、チェックされたコンストラクタ。
> *   `first` : 最後に追加された項目を抽出するセレクタ。
> *   `rest` : 拡張された（残りの）リストを抽出するセレクタ。
> *   `cons?` : `cons` のインスタンスを認識する述語。

図46はこのセクションをまとめたものです。重要な洞察は、`'()` がユニークな値であり、`cons` はリストの値を生成するチェックされたコンストラクタであるということです。さらに、`first`、`rest`、`cons?` は、通常の述語やセレクタに対する単なる別の名前にすぎません。したがって、この章が教えているのは、データを表現する新しい方法ではなく、**「データ定義を定式化する新しい方法」** なのです。

## 8.3 リストを使ったプログラミング (Programming with Lists)

例えば、あなたが友人たちのリストを管理していて、そのリストがあまりにも大きくなったため、ある特定の名前がそのリストに含まれているかどうかを判定するプログラムが必要になったとしましょう。（※注意：ここでは「友人（friend）」という言葉を、現実世界ではなくソーシャルネットワークの意味で使用しています。）
このアイデアを具体化するために、以下の練習問題を考えてみましょう。

> **設計課題（Sample Problem）** 
> あなたは新しい携帯電話の連絡先リストの開発に取り組んでいます。携帯電話の所有者は、さまざまな機会にこのリストを更新し、参照します。差し当たり、あなたはこの連絡先リストを受け取り、その中に `"Flatt"` という名前が含まれているかどうかを判定する関数を設計するタスクを割り当てられました。

この課題の解決策が得られたら、リスト内の任意の名前にマッチする一般的な関数へと一般化します。

先ほど定義した `List-of-names` のデータ定義は、関数が検索する名前のリストを表現するのに適しています。次に、ヘッダー（シグネチャ、目的、スタブ）を作成します。

```racket
;; List-of-names -> Boolean
;; リストに "Flatt" が含まれているかを判定する
(define (contains-flatt? alon)
  #false)
```

関数が受け取る名前のリストを表す引数名として、`a-list-of-names` は良い名前ですが、少し長くて煩雑なため、短く `alon` と略します。

一般的な設計レシピに従い、次に関数の目的を説明する例を作成します。まず、最も単純な入力に対する出力を決定します。それは `'()` です。このリストには文字列が一つも含まれていないため、確実に `"Flatt"` も含まれていません。

```racket
(check-expect (contains-flatt? '()) #false)
```

次に、要素が1つだけのリストを考えます。以下は2つの例です。

```racket
(check-expect (contains-flatt? (cons "Findler" '())) #false)
(check-expect (contains-flatt? (cons "Flatt" '())) #true)
```

最初のケースでは、リストの唯一の項目が `"Flatt"` ではないため、答えは `#false` です。2番目のケースでは、唯一の項目が `"Flatt"` であるため、答えは `#true` です。最後に、より一般的な例を示します。

```racket
(check-expect (contains-flatt? (cons "A" (cons "Flatt" (cons "C" '())))) #true)
```

やはり、リストに `"Flatt"` が含まれているため、答えは `#true` になるはずです。立ち止まって、答えが `#false` にならなければならない一般的な例を作成してください。

一息ついて、プログラムを実行してみましょう。ヘッダーは関数の「ダミー（スタブ）」定義であり、いくつかの例があり、それらはテストに変換されています。最も良いことに、そのうちのいくつかは実際に成功します。誤った理由で成功しているのですが、成功することには変わりありません。ここまで理解できたら、先を読み進めましょう。

第4のステップは、データ定義に一致する関数テンプレートを設計することです。文字列リストのデータ定義には2つの節があるため、関数の本体は2つの節を持つ `cond` 式でなければなりません。2つの条件によって、関数が受け取ったのがどちらの種類のリストであるかを判定します。

```racket
(define (contains-flatt? alon)
  (cond
    [(empty? alon) ...]
    [(cons? alon) ...]))
```

2番目の節では、`(cons? alon)` の代わりに `else` を使用することもできます。

`cond` 式の各節を順に調べることで、テンプレートにさらにヒントを追加できます。具体的には、入力のクラスが複合データ（構造体など）で構成されている場合、設計レシピが各節にセレクタ式を注記することを推奨しているのを思い出してください。今回のケースでは、`'()` には複合データがないため、構成要素はありません。そうでない場合（`cons` の場合）は、リストは1つの文字列と別の文字列リストから構築されているため、テンプレートに `(first alon)` と `(rest alon)` を追加することで、この事実を思い出せるようにします。

```racket
(define (contains-flatt? alon)
  (cond
    [(empty? alon) ...]
    [(cons? alon)
     (... (first alon) ... (rest alon) ...)]))
```

これで、プログラミングの本来のタスクである、設計レシピの第5ステップに移行する準備が整いました。テンプレートから始めて、それぞれの `cond` 節を個別に処理します。

もし `(empty? alon)` が真なら、入力は空のリストです。この場合、関数は結果として `#false` を生成する必要があります。

2番目のケースでは、`(cons? alon)` が真です。テンプレートの注記は、最初の文字列とリストの残りの部分があることを思い出させてくれます。そこで、このカテゴリに該当する例を考えてみましょう。

```racket
(cons "A" (cons ... '()))
```

人は名前を探すとき、最初の項目と `"Flatt"` を比較する必要があります。関数も同様です。この例では、最初の項目は `"A"` であり `"Flatt"` ではないため、比較結果は `#false` になります。代わりに他の例、例えば、

```racket
(cons "Flatt" (cons ... '()))
```

を考えた場合、関数は入力の最初の項目が `"Flatt"` であると判断し、結果として `#true` を返します。これは、`cond` 式の2行目に、リストの最初の名前と `"Flatt"` を比較する式を含めるべきであることを意味します。

```racket
(define (contains-flatt? alon)
  (cond
    [(empty? alon) #false]
    [(cons? alon)
     (... (string=? (first alon) "Flatt") ... (rest alon) ...)]))
```

さらに、比較結果が `#true` になった場合、関数は `#true` を生成する必要があります。

比較結果が `#false` の場合、残りの文字列リストである `(rest alon)` が残されます。明らかに、このケースでは関数は最終的な答えをまだ知ることができません。なぜなら、答えは残りのリストに何が含まれているか（`...` が何を表すか）に依存するからです。言い換えれば、最初の項目が `"Flatt"` でない場合、リストの残りの部分に `"Flatt"` が含まれているかどうかを確認する何らかの方法が必要です。

幸いなことに、私たちには `contains-flatt?` があり、それがまさにその目的に適合します。目的ステートメントによれば、これはリストに `"Flatt"` が含まれているかを判定します。このステートメントは、文字列リスト `l` に対して `(contains-flatt? l)` を呼び出すことで、そのリスト `l` に `"Flatt"` が含まれているかがわかります。同様に、`(contains-flatt? (rest alon))` は、`"Flatt"` が `(rest alon)` のメンバーであるかどうかを判定します。これはまさに私たちが知る必要があることです。

要するに、最後の行には `(contains-flatt? (rest alon))` が含まれるべきです。

```racket
(define (contains-flatt? alon)
  (cond
    [(empty? alon) #false]
    [(cons? alon)
     (... (string=? (first alon) "Flatt")
      ... (contains-flatt? (rest alon)) ...)]))
```

コツは、これら2つの式（「最初の要素がFlattか」と「残りのリストにFlattが含まれるか」）の値を適切に組み合わせることです。前述したように、最初の式が `#true` を生成した場合、リストの残りの部分を検索する必要はありません。しかし、それが `#false` であっても、2番目の式が `#true` を生成する可能性があり、その場合は `"Flatt"` がリストの残りの部分に存在することを意味します。これらすべては、最後の行の最初の式、**または**（`or`）2番目の式が `#true` を返した場合に、`(contains-flatt? alon)` の結果が `#true` になることを示しています。

> **Figure 47: リストの検索**
> 
> ```racket
> ;; List-of-names -> Boolean
> ;; alon に "Flatt" が含まれているかを判定する
> (check-expect (contains-flatt? (cons "X" (cons "Y" (cons "Z" '())))) #false)
> (check-expect (contains-flatt? (cons "A" (cons "Flatt" (cons "C" '())))) #true)
> 
> (define (contains-flatt? alon)
>   (cond
>     [(empty? alon) #false]
>     [(cons? alon)
>      (or (string=? (first alon) "Flatt")
>          (contains-flatt? (rest alon)))]))
> ```

図47は完全な定義を示しています。全体として、この本の最初の章にある定義とそれほど違って見えません。シグネチャ、目的ステートメント、2つの例、および定義で構成されています。この関数定義が、これまでに見てきたものと唯一異なる点は、**自己参照**、すなわち `define` の本体で `contains-flatt?` を参照している点です。しかし、データ定義自体も自己参照的であるため、関数における自己参照もそれほど驚くべきことではありません。

## 8.4 リストの計算 (Computing with Lists)

私たちはまだ BSL を使用しているため、代数の規則（インターメッツォ 1 を参照）から、DrRacket がなくても以下のような式の値を決定する方法がわかります。

```racket
(contains-flatt? (cons "Flatt" (cons "C" '())))
```

プログラマはこの計算がどのように機能するかを直感的に理解しておく必要があるため、この簡単な例の計算ステップを順に追ってみましょう。

図48は最初のステップを示しています。これは通常の代入規則（substitution rule）を使用して適用の値を決定します。結果は条件式（`cond`式）になります。代数の先生が言うように、この関数は段階的な方法で定義されているからです。

> **Figure 48: リストを使った計算、ステップ 1**
> 
> ```racket
> (contains-flatt? (cons "Flatt" (cons "C" '())))
> ==
> (cond
>   [(empty? (cons "Flatt" (cons "C" '()))) #false]
>   [(cons? (cons "Flatt" (cons "C" '())))
>    (or (string=? (first (cons "Flatt" (cons "C" '()))) "Flatt")
>        (contains-flatt? (rest (cons "Flatt" (cons "C" '())))))])
> ```

> **Figure 49: リストを使った計算、ステップ 2**
> 
> ```racket
> ==
> (cond
>   [#false #false]
>   [(cons? (cons "Flatt" (cons "C" '())))
>    (or (string=? (first (cons "Flatt" (cons "C" '()))) "Flatt")
>        (contains-flatt? (rest (cons "Flatt" (cons "C" '())))))])
> ==
> (cond
>   [(cons? (cons "Flatt" (cons "C" '())))
>    (or (string=? (first (cons "Flatt" (cons "C" '()))) "Flatt")
>        (contains-flatt? (rest (cons "Flatt" (cons "C" '())))))])
> ==
> (cond
>   [#true
>    (or (string=? (first (cons "Flatt" (cons "C" '()))) "Flatt")
>        (contains-flatt? (rest (cons "Flatt" (cons "C" '())))))])
> ==
> (or (string=? (first (cons "Flatt" (cons "C" '()))) "Flatt")
>     (contains-flatt? (rest (cons "Flatt" (cons "C" '())))))
> ```

計算は図49へと続きます。`cond` 式の正しい節を見つけるには、条件の値を一つずつ判定する必要があります。`cons` されたリストは空ではないため、最初の条件の結果は `#false` になり、最初の `cond` 節は削除されます。最後に、`cons` されたリストに対する `cons?` は成立するため、2番目の節の条件は `#true` に評価されます。

> **Figure 50: リストを使った計算、ステップ 3**
> 
> ```racket
> ==
> (or (string=? "Flatt" "Flatt")
>     (contains-flatt? (rest (cons "Flatt" (cons "C" '())))))
> ==
> (or #true (contains-flatt? (rest (cons "Flatt" (cons "C" '())))))
> ==
> #true
> ```

ここからは、最終的な結果を得るためのわずか3つの代数ステップです。図50にその3つのステップを示します。
最初のステップは、`first` の法則に基づいて `(first (cons "Flatt" ...))` を `"Flatt"` に評価します。
2番目のステップは、`"Flatt"` が文字列であり `"Flatt"` と等しいことを見出します（`#true`）。
3番目のステップは、`X` が何であれ、`(or #true X)` は `#true` であるという規則を適用します。
