# 8 リスト (Lists)

自己参照データ定義に初めて出会う人が多いでしょう。英語の教師は通常こうした定義を避け、多くの数学の授業でも曖昧にしか扱いません。しかしプログラマは曖昧さを許容できません。仕事には正確さが必要です。

定義は一般に自分自身への参照を複数含むことができますが、この章ではリストを例に、ちょうど1つの自己参照を持つ有用な例から始めます。

リストの導入により、扱える応用例の種類も豊かになります。この章では例を通じて直感を丁寧に築き上げ、次の章で自己参照データ定義を体系的に扱うための設計レシピの改訂を動機づけます。

## 8.1 リストの作成 (Creating Lists)

私たちは日常的にリストを作っています。買い物に行く前には買うもののリストを書きます。朝にはToDoリストを書く人もいます。12月には多くの子供がクリスマスの欲しいものリストを用意します。パーティーを計画する際には招待者のリストを作ります。情報をリストの形で整理することは生活のあらゆる場面にあります。

リストという形の情報が与えられたとき、それをBSLのデータとして表現する方法を明確に学ぶ必要があります。リストが非常に重要であるため、BSLには点 (posn) と同様に、リストの作成と操作のための組み込みサポートがあります。ただし、リストのデータ定義は常に自分で書く必要があります。

まずはリストの作成から始めましょう。

空のリストは `'()` と書きます。

```racket
'()
```

1つの要素を持つリストを作るには `cons` を使います。

```racket
(cons "Flatt" '())
```

2つの要素を持つリスト:

```racket
(cons "Flatt" (cons "C" '()))
```

3つの要素:

```racket
(cons "Flatt" (cons "C" (cons "Felleisen" '())))
```

図44はこれらの構築の様子を箱で示しています。各 `cons` は2つのフィールドを持つ箱です。最初のフィールド（`first`）には要素が入り、2番目のフィールド（`rest`）には残りのリストが入ります。

## 8.2 '() とは何か、cons とは何か (What Is '(), What Is cons)

`'()` は空リストを表す値です。

`cons` は要素とリストを組み合わせて新しいリストを作るコンストラクタです。

対応するセレクタ:

- `(first l)` — リスト `l` の最初の要素を返す
- `(rest l)` — リスト `l` の最初の要素を除いた残りを返す

述語:

- `(empty? l)` — リストが空なら `#true`
- `(cons? l)` — リストが `cons` で構築されたものなら `#true`

例:

```racket
(first (cons "Flatt" '()))
==
"Flatt"

(rest (cons "Flatt" '()))
==
'()

(cons? (cons "Flatt" '()))
==
#true

(empty? '())
==
#true
```

## 8.3 リストを使ったプログラミング (Programming with Lists)

リストを扱う典型的な関数は、以下のような形になります。

```racket
;; List-of-names -> Boolean
;; リストに "Flatt" が含まれているかを判定する
(define (contains-flatt? l)
  (cond
    [(empty? l) #false]
    [(cons? l)
     (or (string=? (first l) "Flatt")
         (contains-flatt? (rest l)))]))
```

例の実行:

```racket
(contains-flatt? (cons "Flatt" '()))
==
(or (string=? "Flatt" "Flatt")
    (contains-flatt? '()))
==
(or #true
    (contains-flatt? '()))
==
#true
```

より長いリストに対する計算も同様に、 `first` と `rest` を繰り返し適用しながら進みます。

## 8.4 リストの計算 (Computing with Lists)

リストに対する計算は、以下のようにステップごとに簡約できます（図48〜50参照）。

```racket
(contains-flatt? (cons "Flatt" (cons "C" '())))
==
(cond
  [(empty? (cons "Flatt" (cons "C" '())) ) #false]
  [else (or (string=? (first (cons "Flatt" (cons "C" '()))) "Flatt")
            (contains-flatt? (rest (cons "Flatt" (cons "C" '())))))])
==
(or (string=? "Flatt" "Flatt")
    (contains-flatt? (cons "C" '())))
==
(or #true
    (contains-flatt? (cons "C" '())))
==
#true
```

このように、リストの操作はすべて `first`、`rest`、`cons`、`empty?`、`cons?` の基本操作に還元されます。
