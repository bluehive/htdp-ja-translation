# 9 自己参照データ定義を使った設計 (Designing with Self-Referential Data Definitions)

一見すると、自己参照データ定義は有限データの定義よりもはるかに複雑に見えるかもしれません。しかし、設計レシピは自己参照データ定義に対してもさらにうまく機能します。新しく加わる部分は、自己参照データ定義が必要になるタイミングを発見するプロセスと、テンプレートをどのように導き出すかに関するものです。

この章では、問題が自己参照データ定義を必要とすることをどのように認識するかを学びます。ここでは `List-of-strings` を例に進めますが、他の原子データのリストも同様に働きます。

## 9.1 指の運動: リスト (Finger Exercises: Lists)

リストに対する関数を書く練習をたくさん行いましょう。

データ定義のおさらい:

```racket
;; A List-of-strings is one of:
;; - '()
;; - (cons String List-of-strings)
```

テンプレート（自己参照データ定義に対する）:

```racket
(define (fun-for-los los)
  (cond
    [(empty? los) ...]
    [else (... (first los) ...
               ... (fun-for-los (rest los)) ...)]))
```

このテンプレートを基に、さまざまな関数を書きます。

例:

- リストの長さを計算する
- リストのすべての要素を大文字にする
- 特定の文字列が含まれるかを判定する（第8章の `contains-flatt?` の一般化）

## 9.2 空でないリスト (Non-empty Lists)

空リストを許容しないデータ定義も有用です。

```racket
;; A NEList-of-temperatures is one of:
;; – (cons CTemperature '())
;; – (cons CTemperature NEList-of-temperatures)
```

この定義では、少なくとも1つの要素が保証されます。`first` は常に安全に呼べます。

対応するテンプレートも変わります（`empty?` のケースがなくなります）。

## 9.3 自然数 (Natural Numbers)

自然数も自己参照データ定義で表せます。

```racket
;; A NaturalNumber is one of:
;; – 0
;; – (add1 NaturalNumber)
```

これにより、再帰的な数値処理（階乗、フィボナッチなど）が自然に書けます。

## 9.4 ロシア人形 (Russian Dolls)

入れ子になった構造（ロシア人形 = マトリョーシカ）も自己参照データ定義の例です。

```racket
(define-struct layer [color doll])

;; A RussianDoll is one of:
;; – String
;; – (make-layer String RussianDoll)
```

`color` と `doll` を使って再帰的に処理します。

## 9.5 リストと World (Lists and World)

World プログラムでもリストは強力です。例えば、複数のオブジェクトを管理する宇宙（複数のUFO、複数のミサイルなど）でリストを使います。

状態はリストとして持ち、毎 tick でリスト全体を更新・描画します。

## 9.6 リストと集合についての注意 (A Note on Lists and Sets)

リストは順序を持ち、重複を許容します。一方、集合は順序がなく、重複がありません。

リストを使って集合のような振る舞いを実現する場合は、明示的に重複を排除するロジックを追加する必要があります。

---

**設計レシピの改訂ポイント（この章で学ぶこと）**:

1. データ定義が自己参照を含むかを確認する。
2. テンプレートに自己参照の再帰呼び出しを自然に配置する。
3. 終了条件（`empty?` など）を明確にする。
4. 再帰呼び出しの結果をどのように組み合わせるかを考える。

これにより、任意の（有限だが）大きさのデータを扱うプログラムを体系的に設計できるようになります。

- リストの合計を求める関数
- リストから奇数だけを抽出する関数
- ロシア人形の最外層の色を返す関数
- 自然数の階乗を計算する関数
