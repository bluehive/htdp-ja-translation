# Intermezzo 1: Beginning Student Language

第I部「固定サイズのデータ」は、BSLを自然言語であるかのように扱いました。言語の「基本的な単語」を紹介し、「単語」を「文」に組み立てる方法を示唆し、これらの「文」の直感的な理解のために代数学の知識に訴えかけました。このような導入はある程度機能しますが、真に効果的なコミュニケーションには、ある程度の形式的な学習が必要です。

多くの点で、第I部のアナロジーは正しいものです。プログラミング言語には語彙と文法があり、プログラマはこれらの要素を**構文 (syntax)** と呼びます。BSLにおける「文」は、式 (expression) または定義 (definition) です。BSLの文法は、これらの句をどのように形成するかを規定します。しかし、文法的に正しい文であっても、すべてが意味を持つわけではありません——英語でもプログラミング言語でも同じです。例えば、「the cat is round」という英語の文は意味を持ちますが、「the brick is a car」は文法的には完全に正しくても意味がありません。文が意味を持つかどうかを判断するには、言語の**意味 (meaning)** を知る必要があります。プログラマはこれを**意味論 (semantics)** と呼びます。

この Intermezzo では、BSLを中学校で馴染みのある算数・代数の言語の拡張であるかのように提示します。結局のところ、計算はこの単純な数学の形式から始まるのですから、この数学とコンピューティングのつながりを理解すべきです。最初の3つの節では、BSLのかなりの部分の構文と意味論を提示します。この新しいBSLの理解に基づき、4つ目の節ではエラーの議論を再開します。残りの節はこの理解を完全な言語に拡張し、最後の節ではテストを表現するためのツールを拡張します。

## BSL の語彙 (BSL Vocabulary)

図39は、BSLの基本的な語彙を紹介し定義しています。語彙は以下のものから構成されます：

- リテラル定数（数値や真偽値など）
- BSLによって意味が与えられている名前（例: `cond` や `+`）
- プログラムが `define` や関数のパラメータを通じて意味を与えることができる名前

> **名前 (name)** または **変数 (variable)** とは、空白や以下の文字を含まない文字の並びです：
> `" , ' ` ( ) [ ] { } | ; #`

- **プリミティブ (primitive)** とは、BSLが意味を割り当てる名前です。例: `+` や `sqrt`。
- **変数 (variable)** とは、事前に意味が割り当てられていない名前です。

**値 (value)** は以下のいずれかです：

- **数 (number)**: `1`, `-1`, `3/5`, `1.22`, `#i1.22`, `0+1i` など（正確数・不正確数、実数・複素数などを含む）
- **ブール値 (Boolean)**: `#true` または `#false`
- **文字列 (String)**: ダブルクォートで囲まれた文字の並び（例: `"hello"`）
- **画像 (Image)**: `circle`, `rectangle` などで生成される画像

（完全な定義は図39を参照。直感的に理解してください。各集合には追加の要素が含まれる場合があります。）

## BSL の文法 (BSL Grammar)

図40は、BSL文法の大部分を示しています。他の言語と比べて非常に単純です。表現力については見た目に惑わされないでください。

文法の読み方：
- 各行の `=` は「〜のいずれかである」を意味します。
- `|` は「または」を意味します。
- `...` は先行するものの繰り返しを意味します。

主要な構文カテゴリ（簡略）:

```
program = def-expr ...
def     = (define name expr)
        | (define (name name ...) expr)
expr    = name
        | value
        | (name expr ...)
        | (cond [expr expr] ... [else expr])
        | (and expr ...)
        | (or expr ...)
```

`define`、`cond`、`else` はキーワードです。これらは名前ではありますが、特別な役割を持つためキーワードと呼ばれます。

## BSL の意味 (BSL Meaning)

DrRacketで式を評価すると、算数と代数の法則を使って**値**を求めます。

これまでのBSLの変種では、値は式の部分集合です。値には Boolean、String、Image、数値などが含まれます。

**意味の法則の例（算術から拡張）**:

```racket
(+ 1 2)
==
3
```

```racket
(+ 1 (+ 1 1) 2)
==
(+ 1 2 2)
==
5
```

条件式の意味:

```racket
(cond
  [(zero? x) 0]
  [else (+ x 1)])
```

最初の条件が真なら対応する式を評価、そうでなければ次の条件へ。`else` は常に真として扱われます。

## 意味とコンピューティング (Meaning and Computing)

BSLの意味は、DrRacket がどのように式をステップごとに簡約していくかを正確に定義します。このプロセスは、算術の計算と本質的に同じです。

## BSL のエラー (BSL Errors)

DrRacket が括弧で囲まれた句がBSLに属さないと判断すると、**構文エラー (syntax error)** を報告します。

文法的に正しくても意味を持たないプログラムは実行時にエラーになります（例: 0での除算、存在しない名前の使用）。

## ブール式 (Boolean Expressions)

BSLには `and` と `or` があります。

```racket
(and (string=? "a" "a") (> 10 9))
==
#true
```

```racket
(or (string=? "a" "b") (= 1 2))
==
#false
```

短絡評価が行われます（必要に応じて後続の式を評価しない）。

## 定数定義 (Constant Definitions)

```racket
(define x 3)
(define y (+ x 1))
(define name "Matthias")
```

定数は式の中で名前として使用できます。

## 構造型定義 (Structure Type Definitions)

```racket
(define-struct posn [x y])
```

これにより `make-posn`、`posn-x`、`posn-y`、`posn?` が自動的に定義されます。

## BSL のテスト (BSL Tests)

```racket
(check-expect (f 1) 2)
(check-expect (f 2) 4)

(check-within (g 1) 3.14 0.01)
(check-error (h 0) "division by zero")
```

テストはプログラムの一部として書くことができます。

## BSL のエラーメッセージ (BSL Error Messages)

DrRacket は親切なエラーメッセージを出します。主要なカテゴリ:

- 構文エラー（括弧の不一致、キーワードの誤用）
- 実行時エラー（未定義の名前、型エラー、0除算など）
- テスト失敗

各エラーメッセージは問題の場所と可能な原因を示唆します。

（詳細なエラーメッセージ一覧は本書の該当節を参照してください。）
