リストは本書でも、教育用言語の基盤である Racket でも重要な役割を果たす。プログラム設計では、リストが第一原理からどのように構築されるかを理解することが肝心であり、それがプログラムの作り方を導く。一方、リストを日常的に扱うには、The list Function（list 関数）で導入した `list` 関数のような簡潔な記法が必要になる。

> **注:** 言語レベルを BSL+ 以上に設定すること。

1950年代後半以来、Lisp 系言語はさらに強力なリスト生成の道具の対——引用（quotation）と反引用（anti-quotation）——を備えてきた。いまでは多くのプログラミング言語がこれをサポートし、PHP という Web ページ設計言語がこの発想を商用の世界に持ち込んだ。

このインターメッツォでは、このクォート機構の一端を味わう。あわせて、クォートと密接に結び付いたデータの一種である *シンボル*（symbol）も紹介する。ここでの導入は非公式で単純な例を用いるが、本書の残りでは、ほぼ現実的な変形例でこの発想の力を示す。そうした例でつまずいたら、このインターメッツォに戻ってほしい。

## Quote（クォート）

クォートは、大きなリストを簡単に書き下すための短縮記法である。おおざっぱに言えば、`list` 関数で構築するリストは、リストをクォートするだけでさらに簡潔に書ける。逆に、クォートされたリストは `list` による構築の略記である。

技術的には、`quote` は Intermezzo 1: Beginning Student Language の精神に沿った複合文のキーワードで、次のように使う：`(quote (1 2 3))`。
DrRacket はこの式を `(list123)` に変換する。ここで、「なぜ `quote` を略記と呼ぶのか、クォートした式の方が変換結果より複雑に見えるではないか」と思うかもしれない。鍵は、`'` が `quote` の短縮形だという点にある。短い例を挙げる：

```racket
> '(1 2 3)
(list 1 2 3)
> '("a" "b" "c")
(list "a" "b" "c")
> '(#true "hello world" 42)
(list #true "hello world" 42)
```

見てのとおり、`'` を使うと約束どおりのリストができる。
もし `(list123)` の意味を忘れていれば、The list Function（list 関数）を読み直してほしい。そこでは、このリストが
`(cons1(cons2(cons3'())))` の短縮だと説明されている。

ここまでは `quote` は `list` に対する小さな改善に見えるが、次を見てほしい：

```racket
> '(("a" 1)    ("b" 2)    ("d" 4))
(list (list "a" 1) (list "b" 2) (list "d" 4))
```

`'` を使えば、リストも入れ子のリストも構築できる。

`quote` の働きを理解するには、与えられた形をたどる関数だと想像するとよい。`'` が素のデータ——数、文字列、ブール値、または画像——に出会うと、それは消える。開き括弧 `(` の前にあるときは、括弧の右に `list` を挿入し、`(` と閉じ括弧 `)` のあいだのすべての項目に `'` を付ける。たとえば、

> `'(1 2 3)` は `(list '1 '2 '3)` の短縮である

すでに知っているように、数の前の `'` は消えるので、残りは簡単だ。入れ子リストを作る例は次のとおり：

> `'(("a" 1) 3)` は `(list '("a" 1) '3)` の短縮である

この例を続けると、先頭位置の略記を展開する：

> `(list '("a" 1) '3)` は `(list (list '"a" '1) 3)` の短縮である

この例の締めくくりは読者に任せる。

**演習 231.** 次の式から `quote` をなくし、`list` で書き直せ：

- `'(1"a"2#false3"c")`
- `'()`
- および次の表のような形：
`'(("alan" 1000) ("barb" 2000) ("carl" 1500))`

さらに、必要なところでは `list` を `cons` に置き換えよ。

## Quasiquote and Unquote（準クォートとアンクォート）

前節で `'` と `quote` の利点は納得できたはずだ。なぜ本書が `quote` をいまになって導入し、最初から使わなかったのか不思議に思うかもしれない。リストを含むテストケースの記述や、大量のデータを扱う際に大いに役立ちそうに見える。しかし、良いものには必ず驚きが付きものであり、`quote` も例外ではない。

プログラム設計に関して言えば、初心者がリストをクォートされた値や `list` で構築した値として考えるのは誤解を招きやすい。プログラムを段階的に作るうえで、`cons` によるリスト構築の方が、背後の構築を隠す `quote` のような短縮記法よりはるかに啓発的である。リスト処理関数の設計で行き詰まったら、いつでも `cons` で考えることを忘れないでほしい。

では、`quote` の背後に潜む実際の驚きに移ろう。定義領域に次の定数定義がひとつあるとする：

```racket
(define x 42)
```

このプログラムを実行し、対話領域で次を試す場面を想像してほしい：

```racket
'(40 41 x 43 44)
```

どんな結果を期待するだろうか。ストップ！ しばらく、上で述べた `'` の規則を当てはめてみよ。

実験の結果は次のとおり：

```racket
> '(40 41 x 43 44)
(list 40 41 'x 43 44)
```

ここで重要なのは、DrRacket が表示するのは*値*だということだ。リスト上のものはすべて値であり、`'x` もそうである。これはこれまでに見たことのない値、すなわち*シンボル*（Symbol）である。我々の目的では、シンボルは変数名のように見えるが、`'` で始まり、かつシンボルは*値*である。変数は値を*表す*だけで、それ自体は値ではない。シンボルは文字列と似た役割を果たし、記号的な情報をデータとして表現するのに非常に適している。Intertwined Data（絡み合ったデータ）がその使い方を示す。いまは、シンボルをまた別のデータの一形態として受け入れておけばよい。

シンボルの考え方を定着させるために、二つ目の例を考える：

```racket
'(1 (+ 1 1) 3)
```

この式が `(list123)` を構築すると期待するかもしれない。しかし、`'` を展開する規則を使うと、次のようになることがわかる：

> `'(1 (+ 1 1) 3)` は `(list '1 '(+ 1 1) '3)` の短縮である

そして、このリストの第二項目の前の `'` は消えない。代わりに、別のリストの構築を略記するので、例全体は次のようになる：

```racket
(list 1 (list '+ 1 1) 3)
```

つまり `'+` は `'x` と同じくシンボルである。後者が変数 `x` と無関係であるのと同様に、前者も BSL+ に付属する関数 `+` と直接の関係はない。
とはいえ、`'+` が関数 `+` の優雅なデータ表現になり得ること、そして `'(+11)` が `(+11)` のデータ表現になり得ることは想像できるはずだ。Intertwined Data（絡み合ったデータ）がこの発想を取り上げる。

場合によっては、入れ子のリストを作りたくないこともある。クォートされたリストの中に*本当の*式を置き、リスト構築の過程でその式を*評価*したい。そのような場合に使うのが *quasiquote*（準クォート）であり、`quote` と同様、複合文のキーワードにすぎない：`(quasiquote (1 2 3))`。そして `quote` と同様、準クォートにも短縮形があり、キーボードの「もう一方の」単一引用符キーである `` ` `` 文字である。

一見すると、`` ` `` はリストを構築する点で `'` と同じように振る舞う：

```racket
> `(1 2 3)
(list 1 2 3)
> `("a" "b" "c")
(list "a" "b" "c")
> `(#true "hello world" 42)
(list #true "hello world" 42)
```

`` ` `` の最良の点は、*アンクォート*（unquote）にも使えること、つまり準クォートされたリストの内側で、本来のプログラミング言語への脱出を要求できることだ。上の例でこの考えを示そう：

```racket
> `(40 41,x 43 44)
(list 40 41 42 43 44)
> `(1,(+ 1 1) 3)
(list 1 2 3)
```

上と同様、最初の対話は定義領域に
`(definex42)` があることを前提とする。この構文を理解する最善の方法は、`` ` `` と `,` の短縮形ではなく、実際のキーワードで見ることだ：

```racket
(quasiquote (40 41 (unquote x) 43 44))
(quasiquote (1 (unquote (+ 1 1)) 3))
```

準クォートとアンクォートの形を展開する規則は、`quote` の規則に一つの規則を足したものである。`` ` `` が括弧の前にあるとき、それはその括弧と対応する閉じ括弧のあいだのすべての部分に分配される。基本的なデータの隣にあるときは消える。変数名の前にあるときはシンボルが得られる。そして新しい規則は、`` ` `` の直後に unquote が続くとき、両方の文字が消える、というものである：

> `` `(1,(+ 1 1) 3) `` は `(list `1 `,(+ 1 1) `3)` の短縮である

そして

> `(list `1 `,(+ 1 1) `3)` は `(list 1 (+ 1 1) 3)` の短縮である

こうして、上で見たように `(list123)` が得られる。

ここから Web ページの生成まではもう一歩だ。そう、正しく読んだ——Web ページだ！ 原理的には、Web ページは HTML と CSS というプログラミング言語で記述される。しかし、誰も HTML プログラムを直接書き下しはしない。代わりに、Web ページを*生成する*プログラムを設計する。驚くことではないが、BSL+ でもそのような関数を書ける。図83に単純な例がある。

## Unquote Splice（アンクォート・スプライス）

短縮形の展開中に quasiquote が unquote に出会うと、両者は互いに消滅する：

> `` `(tr,(make-row '(3 4 5))) `` は `(list 'tr (make-row (list 3 4 5)))` の短縮である

したがって、`make-row` が何を生成しても、それがリストの第二項目になる。とくに、`make-row` がリストを生成すれば、そのリストがリストの第二項目になる。`make-row` が与えられた数のリストを文字列のリストに変換するなら、結果は次のとおり：

```racket
(list 'tr (list "3" "4" "5"))
```

しかし場合によっては、そのような入れ子リストを外側のリストに*スプライス*（継ぎ足し）して、この実行例では次のようにしたい：

```racket
(list 'tr "3" "4" "5")
```

この小さな問題を解く一つの方法は、`cons` に戻ることだ。つまり、`cons` を `quote`、`quasiquote`、`unquote` と混ぜて使う。結局のところ、これらの文字はすべて `cons` で作られたリストの短縮にすぎない。この例で望む結果を得るには、次が必要である：

```racket
(cons 'tr (make-row '(3 4 5)))
```

結果が `(list'tr"3""4""5")` になることを自分で確認せよ。

この状況は実践でかなりよく起きるので、BSL+ はリスト生成のための短縮機構をもう一つサポートしている：`,@`。キーワード形では *unquote-splicing* としても知られる。この形を使えば、入れ子リストを囲むリストにスプライスするのは簡単だ。たとえば、

```racket
`(tr,@(make-row '(3 4 5)))
```

は次に変換される：

```racket
(cons 'tr (make-row '(3 4 5)))
```

これはまさに、この例で必要なものである。

次に、入れ子リスト表現で HTML の表を作る問題を考える。次は各行に 4 つのセルを持つ 2 行の表である：

> `'(table((border"1"))(tr(td"1")(td"2")(td"3")(td"4"))(tr(td"2.8")(td"-1.1")(td"3.4")(td"1.3")))`

最初の入れ子リストは、表の各セルの周りに細い枠線を描くよう HTML に指示する。他の二つの入れ子リストは各々 1 行を表す。

実際には、任意に広い行と任意の行数を持つような表を作りたい。いまは最初の問題だけを扱い、数のリストを HTML の行に変換する関数が必要である：

```racket
; List-of-numbers ->... nested list...
; creates a row for an HTML table from l
(define (make-row l)
  (cond
    [(empty? l) '()]
    [else (cons (make-cell (first l))
                (make-row (rest l)))]))

; Number ->... nested list...
; creates a cell for an HTML table from a number
(define (make-cell n)
  `(td,(number->string n)))
```

例を追加する代わりに、DrRacket の対話領域でこれらの関数の振る舞いを探る：

```racket
> (make-cell 2)
(list 'td "2")
> (make-row '(1 2))
(list (list 'td "1") (list 'td "2"))
```

これらの対話は、セルと行を表すリストの生成を示す。

このような行リストを、HTML 表表現の実際の行に変えるには、`'tr` で始まるリストにそれらをスプライスする必要がある：

```racket
; List-of-numbers List-of-numbers ->... nested list...
; creates an HTML table from two lists of numbers
(define (make-table row1 row2)
  `(table ((border "1"))
          (tr,@(make-row row1))
          (tr,@(make-row row2))))
```

この関数は二つの数のリストを受け取り、HTML 表の表現を作る。`make-row` でリストをセル表現のリストに変換する。`,@` でそれらのリストを表のテンプレートにスプライスする：

```racket
> (make-table '(1 2 3 4 5) '(3.5 2.8 -1.1 3.4 1.3))
(list 'table (list (list 'border "1")) '....)
```

この `make-table` の適用は、人々が Web ページを手作業で作らずプログラムで作るもう一つの理由を示唆する。

> **注:** ドットは出力の一部ではない。

**ヒント** 文字列のリストから順位を決める関数を設計することもできるが、ここでは表の作成に集中してほしい。そこで次の関数を提供する：

```racket
(define (ranking los)
  (reverse (add-ranks (reverse los))))

(define (add-ranks los)
  (cond
    [(empty? los) '()]
    [else (cons (list (length los) (first los))
                (add-ranks (rest los)))]))
```

これらの関数を使う前に、シグネチャと目的文を付けよ。その後、DrRacket の対話で動作を探れ。Accumulators（蓄積子）は設計レシピを拡張し、`ranking` や `add-ranks` より単純な順位計算関数を作る方法を示す。
