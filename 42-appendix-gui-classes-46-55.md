## editor-stream-out%

```
+----------------------------------+
| classeditor-stream-out%: class? |
+----------------------------------+
| superclass: object%              |
+----------------------------------+
```

`editor-stream-out%` オブジェクトは、エディタ情報をファイルやその他の出力ストリーム（クリップボードなど）へ書き出すために使います。

```
+------------------------------------------+
| [constructor]                            |
|                                          |
| (make-object editor-stream-out% base)    |
| → (is-a?/c editor-stream-out%)           |
| base: (is-a?/c editor-stream-out-base%) |
+------------------------------------------+
```

出力ストリームのベース——たとえば `editor-stream-out-bytes-base%` オブジェクト——を `base` として渡さなければなりません。

```
+-------------------------------------------------+
| [method]                                        |
|                                                 |
| (send an-editor-stream-out jump-to pos) → void? |
| pos: exact-nonnegative-integer?                |
+-------------------------------------------------+
```

ストリーム内の指定位置へジャンプします。

```
+--------------------------------------------+
| [method]                                   |
|                                            |
| (send an-editor-stream-out ok?) → boolean? |
+--------------------------------------------+
```

ストリームが書き込み可能なら #t、そうでなければ #f を返します。不正なストリームへの書き込みは効果がありません。

```
+---------------------------------------------------+
| [method]                                          |
|                                                   |
| (send an-editor-stream-out pretty-finish) → void? |
+---------------------------------------------------+
```

ストリームが改行で終わることを保証します。このメソッドは `write-editor-global-footer` から呼ばれます。

```
+--------------------------------------------------+
| [method]                                         |
|                                                  |
| (send an-editor-stream-out pretty-start) → void? |
+--------------------------------------------------+
```

ファイル形式を識別する「コメント」をストリームへ書き込みます。このメソッドは `write-editor-global-header` から呼ばれます。

```
+-------------------------------------+
| [method]                            |
|                                     |
| (send an-editor-stream-out put n v) |
| → (is-a?/c editor-stream-out%)      |
| n: exact-nonnegative-integer?      |
| v: bytes?                          |
| (send an-editor-stream-out put v)   |
| → (is-a?/c editor-stream-out%)      |
| v: bytes?                          |
| (send an-editor-stream-out put v)   |
| → (is-a?/c editor-stream-out%)      |
| v: exact-integer?                  |
| (send an-editor-stream-out put v)   |
| → (is-a?/c editor-stream-out%)      |
| v: real?                           |
+-------------------------------------+
```

`v` を書き込みます。あるいは `v` の `n` バイトを書き込みます。

バイト列 `v` とともに `n` を渡した場合は、後で `get-unterminated-bytes` を使って読み戻してください。後で読みやすい形でバイト列を書き出す推奨方法はこれです。読み戻すときは `get-unterminated-bytes` を使います。

`n` を渡さず、かつ `v` がバイト列の場合は、歴史的な理由により、実際に書き込まれるバイト数には `#\nul` 終端が含まれます。そのため後で読むときは `get-unterminated-bytes` ではなく `get-bytes` を使ってください。

`v` が `real?` であって `exact-integer?` でない場合、書き込み処理の一部として非正確数へ変換されます。

```
+------------------------------------------+
| [method]                                 |
|                                          |
| (send an-editor-stream-out put-fixed v)  |
| → (is-a?/c editor-stream-out%)           |
| v: (integer-in -9999999999 99999999999) |
+------------------------------------------+
```

固定サイズの整数をストリームへ書き込みます。通常、数値はバイト数が可変の形式で書かれるため、このメソッドが必要になることがあります。一時的にストリームへ `0` を書き、さらにデータを書き、その後戻って `0` を別の数に書き換える、といった処理では固定サイズの数が必要です。

`put-fixed` でストリームに書いた数は、`get-fixed-exact` または `get-fixed` で読み取らなければなりません。

```
+------------------------------------------------+
| [method]                                       |
|                                                |
| (send an-editor-stream-out put-unterminated v) |
| → (is-a?/c editor-stream-out%)                 |
| v: bytes?                                     |
+------------------------------------------------+
```

`(bytes-length v)` と `v` を渡して `put` を呼ぶのと同じです。

```
+---------------------------------------------------------------+
| [method]                                                      |
|                                                               |
| (send an-editor-stream-out tell) → exact-nonnegative-integer? |
+---------------------------------------------------------------+
```

現在のストリーム位置を返します。

## editor-wordbreak-map%

```
+-------------------------------------+
| classeditor-wordbreak-map%: class? |
+-------------------------------------+
| superclass: object%                 |
+-------------------------------------+
```

`editor-wordbreak-map%` オブジェクトは、`text%` オブジェクトと組み合わせて、既定のワードブレーク関数の語分割基準を指定するために使います。`set-wordbreak-map`、`get-wordbreak-map`、`find-wordbreak`、`set-wordbreak-func` も参照してください。

グローバルオブジェクト `the-editor-wordbreak-map` が自動的に作成され、すべての `text%` オブジェクトの既定マップとして使われます。

ワードブレークオブジェクトは、各文字からシンボルのリストへの写像を実装します。リストの要素として許されるシンボルは次のとおりです。

- 'caret
- 'line
- 'selection
- 'user1
- 'user2

文字の値にフラグが含まれていると、対応する理由でブレークを探すとき、その文字は語を分割しません。たとえば `'caret` がある場合、キャレット移動用の語としてはその文字は非分割文字です（連続する非分割文字の列が 1 語になります）。

```
+---------------------------------------------------------------+
| [constructor]                                                 |
|                                                               |
| (new editor-wordbreak-map%) → (is-a?/c editor-wordbreak-map%) |
+---------------------------------------------------------------+
```

すべての ASCII 英数字は `'(caret line selection)` で初期化されます。`-` を除く他の ASCII 非空白文字は `'(line)` で初期化されます。すべての ASCII 空白文字と `-` は `null` で初期化されます。

```
+---------------------------------------------------------+
| [method]                                                |
|                                                         |
| (send an-editor-wordbreak-map get-map char)             |
| → (listof (or/c 'caret 'line 'selection 'user1 'user2)) |
| char: char?                                            |
+---------------------------------------------------------+
```

`char` の写像値を取得します。詳細は `editor-wordbreak-map%` を参照してください。

```
+---------------------------------------------------------------+
| [method]                                                      |
|                                                               |
| (send an-editor-wordbreak-map set-map                         |
| char: char?                                                  |
| value: (listof (or/c 'caret 'line 'selection 'user1 'user2)) |
+---------------------------------------------------------------+
```

`char` の写像値を `value` に設定します。詳細は `editor-wordbreak-map%` を参照してください。

## editor<%>

```
+---------------------------------+

+---------------------------------+
+---------------------------------+
```

`editor<%>` インタフェースは `text%` と `pasteboard%` によって実装されます。

```
+--------------------------------------------+
| [method]                                   |
|                                            |
| (send an-editor add-canvas canvas) → void? |
| canvas: (is-a?/c editor-canvas%)          |
+--------------------------------------------+
```

このエディタを表示しているキャンバスのリストへキャンバスを追加します（`get-canvases` を参照）。

通常、このメソッドは editor-canvas% の set-editor からのみ呼ばれます。

```
+------------------------------------------+
| [method]                                 |
|                                          |
| (send an-editor add-undo undoer) → void? |
| undoer: (-> any)                        |
+------------------------------------------+
```

アンドゥ手続きをエディタのアンドゥスタックへ追加します。現在アンドゥ実行中なら、その手続きはリドゥスタックへ追加されます。システムがエディタの変更をアンドゥ（またはリドゥ）するとき、かつこの手続きがアンドゥ（またはリドゥ）スタックの先頭であるときに呼ばれます。

エディタインスタンスはアンドゥスタックなしで作成されるため、set-max-undo-history でアンドゥスタックのサイズを変更しない限り、アンドゥ操作は記録されません。

システムは挿入・削除・フォント変更などの組み込みエディタ操作を取り消すアンドゥ記録を自動的にインストールします。アンドゥ手続きをインストールするのは、状態を維持する必要があるとき、または組み込みでない操作を扱うときに限ってください。たとえば、ユーザーがペーストボード上のスニップにラベルを付けられるプログラムでは、ラベル変更を元に戻すアンドゥ手続きをインストールすべきです。これにより、ユーザーがスニップのラベルを変えたあと（標準メニューバーから）Undo を選ぶと、期待どおりラベルが戻ります。対照的に、ユーザーがドラッグでスニップを動かしたときにはアンドゥ手続きを入れる必要はありません。システムが適切なアンドゥを自動で入れるためです。

アンドゥ手続きが戻ると、その手続きはエディタのアンドゥ（またはリドゥ）スタックから取り出されます。戻り値が真なら、同じアンドゥ（またはリドゥ）ステップの一部として次のアンドゥ手続きも実行されます。取り消される動作がもともと begin-edit-sequence と end-edit-sequence のシーケンスの一部として行われた場合、アンドゥ手続きは真を返すべきです。暗黙にシーケンスの一部だった場合も戻り値は真であるべきです。前の例を拡張すると、ラベル変更がスニップ再配置の移動と対になっているなら、ラベル変更のアンドゥ手続きは move の呼び出しのあとにエディタへ追加し、呼ばれたときに #t を返すべきです。その結果、ラベル変更の取り消しの直後に移動も取り消されます（逆順が必要なら begin-edit-sequence と end-edit-sequence で明示シーケンスを作ってください）。

システムは（他のメソッド呼び出しに応じて）このメソッドを呼ばずにエディタへアンドゥ手続きを追加します。

```
+--------------------------------------+
| [method]                             |
|                                      |
| (send an-editor adjust-cursor event) |
| → (or/c (is-a?/c cursor%) #f)        |
| event: (is-a?/c mouse-event%)       |
+--------------------------------------+
```

仕様:
エディタの表示で使うカーソルを取得します。戻り値が #f の場合、既定のカーソルが使われます。

set-cursor も参照してください。

デフォルト実装:
set-cursor で上書きカーソルがインストールされていれば、そのカーソルを返します。

そうでなければ、イベントがドラッグイベントであり、エディタ内のスニップがフォーカスを持ち、そのスニップの adjust-cursor メソッドがカーソルを返す場合、そのカーソルを返します。

そうでなければ、カーソルがスニップ上にあり、そのスニップの adjust-cursor メソッドがカーソルを返す場合、そのカーソルを返します。

そうでなければ、set-cursor でカーソルがインストールされていれば、そのカーソルを返します。

そうでなければ、カーソルがエディタのクリックバック領域上にある場合、矢印カーソルを返します。

最後に、上記のいずれにも当てはまらない場合、既定のカーソルを返します。テキストエディタでは既定は I ビーム、ペーストボードエディタでは矢印です。

on-display-size に応じて set-max-width を自動呼び出しするかどうかを有効／無効にするか、オートラップの状態を取得します。テキストエディタでは、エディタを表示するキャンバス（複数ある場合は最も広いもの）に内容が収まるよう折り返す効果があります。ペーストボードエディタでは、「オートラップ」はペーストボードの領域をキャンバス表示に合わせて切り詰めるだけです。

オートラップは初期状態では無効です。

```
+-------------------------------------+
| [method]                            |
|                                     |
| (send an-editor begin-edit-sequence |
| undoable?: any/c = #t              |
| interrupt-streak?: any/c = #t      |
+-------------------------------------+
```

仕様:
begin-edit-sequence と end-edit-sequence メソッドは、一連のエディタ変更を囲み、結果を一度に表示するために使います。コマンドは任意の深さにネストできます。これらの関数を使うと変更の表示を大幅に高速化できます。

エディタが他のエディタを含む場合、メインエディタで begin-edit-sequence と end-edit-sequence を使うとサブエディタの一部の変更も囲みますが、サブエディタ側が変わったときにはサブエディタで begin-edit-sequence と end-edit-sequence を呼ぶほど効果的ではありません。

refresh-delayed? と in-edit-sequence? も参照し、編集シーケンスと再描画要求については「エディタとスレッド」を参照してください。

undoable? フラグが #f の場合、シーケンス内の変更は undo メソッドでは元に戻せません。アンドゥの詳細は下記を参照。ネストしたシーケンスでは、undoable? フラグが有効なのは最も外側の begin-edit-sequence だけです。なお text% オブジェクトでは、文字挿入版の insert がシーケンス単位のアンドゥグループ化と干渉します。

interrupt-streak? フラグが #f で、かつシーケンスが最外の場合、シーケンスの前後の特別な動作は連続した動作として扱われます。たとえばシーケンス直前と直後の kill はクリップボード上で連結されます。

アンドゥの詳細: undoable? が #f の振る舞いは、アンドゥログに項目を追加しないことで実装されます。たとえば、エディタに a を挿入し、b を挿入し、アンドゥ不可の編集シーケンスを開始して a を赤くし、シーケンスを終えると、アンドゥは b を削除し、赤い a が残ります。

別の例として、次の対話では undo は b ではなく a を削除します。

```racket
> (define t (new text%))
> (begin   (send t set-max-undo-history 'forever)   (send t insert "a")   (send t insert "b")   (send t begin-edit-sequence #f #f)   (send t insert "c" 0 0)   (send t end-edit-sequence)   (send t get-text))
"cab"
> (begin   (send t undo)   (send t get-text))
"cb"
```

    (make-object editor-stream-in% editor-stream-in-bytes-base))
  (read-editor-global-header editor-stream-in)
  (send editor read-from-file editor-stream-in)
  (read-editor-global-footer editor-stream-in)
  editor)
```

snip にキーボードフォーカスを与えようとします。snip が #f なら、エディタローカルのフォーカスを持つスニップからキャレットを取り上げます。

キーボードフォーカスが snip へ移り、かつエディタが実際のキーボードフォーカスを持つ場合、そのスニップの own-caret メソッドが #t で呼ばれます。

フォーカス設定の領域は次のいずれかです。

- 'immediate — エディタ内だけでフォーカス所有者を設定
- 'display — このエディタまたは新しいフォーカス所有者が、このエディタの表示内のエディタ間でキーボードフォーカスを得る（埋め込みエディタの場合）
- 'global — このエディタまたは新しいフォーカス所有者が、エディタのフレーム内の全要素間でキーボードフォーカスを得る

ファイルの sha1 を更新するときに呼ばれます（enable-sha1 が呼ばれている場合のみ）。このメソッドが #f を返すと sha1 は更新されず、get-file-sha1 の結果も変わりません。

is-sha1-enabled? も参照してください。

Added in version 1.50 of package `gui-lib`.

```
+-------------------------------------------------+
| [method]                                        |
|                                                 |
| (send an-editor use-file-text-mode) → boolean?  |
| (send an-editor use-file-text-mode on?) → void? |
| on?: any/c                                     |
+-------------------------------------------------+
```

ファイルを 'text または 'binary モード（open-input-file の #:mode と同様）で保存するかを制御するブールを取得または設定します。このフラグは format が 'text または 'text-force-cr のときだけ参照されます。形式については load-file を参照してください。

この設定は on-save-file が呼ばれたあとの save-file によって参照されます。

このメソッドのオーバーライドは、内部ブール値の変化を検出する確実な方法です。

```
+----------------------------------------------------------+
| [method]                                                 |
|                                                          |
| (send an-editor write-footers-to-file stream) → boolean? |
| stream: (is-a?/c editor-stream-out%)                    |
+----------------------------------------------------------+
```

write-headers-to-file を参照してください。

```
+----------------------------------------------------------+
| [method]                                                 |
|                                                          |
| (send an-editor write-headers-to-file stream) → boolean? |
| stream: (is-a?/c editor-stream-out%)                    |
+----------------------------------------------------------+
```

仕様:
エディタがファイルへ保存されるときに呼ばれます。エラーがなければ #t、あれば #f を返します。カスタムヘッダデータを追加するにはこのメソッドをオーバーライドしますが、継承メソッドが独自の追加ヘッダを書けるよう常に呼んでください。

ヘッダ項目を書くには、整数用のボックスを渡して begin-write-header-footer-to-file を呼びます。次にヘッダデータを書き、ボックスに入った整数を渡して end-write-header-footer-to-file を呼んで終えます。手順を誤るとファイルが壊れます。

デフォルト実装:
何もしません。

```
+--------------------------------------------------+
| [method]                                         |
|                                                  |
| (send an-editor write-to-file stream) → boolean? |
| stream: (is-a?/c editor-stream-out%)            |
+--------------------------------------------------+
```

現在のエディタ内容を与えられたストリームへ書き込みます。エラーがなければ #t、あれば #f を返します。

エディタのスタイルリストがすでにストリームへ書かれている場合、再書き込みはしません。代わりに、エディタ内容は以前に書かれたスタイルリストを共有することを示します。この共有はストリームを後で読むときに再構築されます。

write-to-file を使ってエディタストリームへ書くには、一見明らかでない手続きが必要です。write-editor-global-header と write-editor-global-footer の呼び出しが write-to-file の前後を囲まなければならず、これらのメソッドや read-editor-global-header / read-editor-global-footer で一度に扱えるストリームはひとつだけです。「ファイル形式」も参照してください。

完全な例として、次を考えてください。

```racket
(define (serialize-text text)
  (define editor-stream-out-bytes-base
    (new editor-stream-out-bytes-base%))
  (define editor-stream-out
    (make-object editor-stream-out% editor-stream-out-bytes-base))
  (write-editor-global-header editor-stream-out)
  (send text write-to-file editor-stream-out)
  (write-editor-global-footer editor-stream-out)
  (send editor-stream-out-bytes-base get-bytes))
```

## frame%

```
+----------------------+---------------------+
| classframe%: class? |                     |
+----------------------+---------------------+
| superclass: object%  |                     |
| extends:             | top-level-window<%> |
+----------------------+---------------------+

フレームはトップレベルのコンテナウィンドウです。タイトルバー（フレームのラベルを表示）、オプションのメニューバー、オプションのステータス行を持ちます。

```
+--------------------------------------------------------------------------------+
| [constructor]                                                                  |
|                                                                                |
| (new frame%                                                                    |

| x: (or/c position-integer? #f) = #f                                           |
| y: (or/c position-integer? #f) = #f                                           |
| style: (listof (or/c 'no-resize-border 'no-caption 'no-system-menu            |
| 'hide-menu-bar 'toolbar-button 'float 'metal 'fullscreen-button                |
| 'fullscreen-aux)) = null                                                       |
| (listof (or/c 'no-resize-border 'no-caption                                    |
| 'no-system-menu 'hide-menu-bar                                                 |
| 'toolbar-button 'float 'metal                                                  |
| 'fullscreen-button 'fullscreen-aux))                                           |

| border: spacing-integer? = 0                                                  |
| spacing: spacing-integer? = 0                                                 |
| alignment: (list/c (or/c 'left 'center 'right) (or/c 'top 'center 'bottom)) = |
| '(center top)                                                                  |
| (list/c (or/c 'left 'center 'right)                                            |
| (or/c 'top 'center 'bottom))                                                   |
| min-width: (or/c dimension-integer? #f) = #f                                  |
| min-height: (or/c dimension-integer? #f) = #f                                 |
| stretchable-width: any/c = #t                                                 |
| stretchable-height: any/c = #t                                                |
|                                                                                |
| ```racket                                                                      |
| (listof (or/c 'no-resize-border 'no-caption                                    |
|               'no-system-menu 'hide-menu-bar                                   |
|               'toolbar-button 'float 'metal                                    |
|               'fullscreen-button 'fullscreen-aux))                             |
| ```                                                                            |
|                                                                                |
| ```racket                                                                      |
| (list/c (or/c 'left 'center 'right)                                            |
|         (or/c 'top 'center 'bottom))                                           |
| ```                                                                            |
+--------------------------------------------------------------------------------+
```

label 文字列はフレームのタイトルバーに表示されます。フレームのラベルを変更すると（set-label を参照）、タイトルバーが更新されます。

parent 引数は #f、または既存のフレーム／ダイアログです。Windows では parent が #f でなければ新しいフレームは常に親の上に表示されます。Windows と Unix（多くのウィンドウマネージャ）では、親がアイコン化されるとフレームもアイコン化されます。

parent が #f なら、新しいフレームのイベントスペースは current-eventspace が決める現在のイベントスペースです。そうでなければ親のイベントスペースが使われます。

width または height が #f でなければ、最小サイズより大きい前提でフレームの初期サイズ（ピクセル）を指定します。そうでなければ最小サイズが使われます。

x または y が #f でなければフレームの初期位置を指定します。そうでなければ位置は自動選択されます（作成に応じてフレームとダイアログをタイル配置）。

style フラグは、一部のプラットフォームでフレームの見た目を調整します。

- 'no-resize-border — ウィンドウ周りのリサイズ可能な境界を省略（Windows、Unix）、ウィンドウのリサイズ能力を省略（Mac OS）、または右下のグローボックスを省略（古い Mac OS）
- 'no-caption — フレームのタイトルバーを省略（Windows、Mac OS、Unix）
- 'no-system-menu — システムメニューを省略（Windows）
- 'toolbar-button — フレームのタイトルバーにツールバーボタンを含める（Mac OS 10.6 以前）。クリックは on-toolbar-button-click を引き起こす
- 'hide-menu-bar — フレームがアクティブなときメニューバーとドックを隠す（Mac OS）、またはウィンドウマネージャにフルスクリーン化を依頼する（Unix）
- 'float — フレームを他の非フロートウィンドウすべての前面に保つ（Windows、Mac OS、Unix）
- 'metal — 無視（以前は Mac OS 向け）
- 'fullscreen-button — フルスクリーン用タイトルバーボタンを含める（Mac OS 10.7 以降）
- 'fullscreen-aux — フルスクリーン中の別フレームに同伴できる（Mac OS 10.7 以降）

フレームが表示されていなくても、作成時に少数の通知イベントがキューに入ることがあります。そのため新しいフレームの資源（メモリなど）は、いくつかのイベントが処理されるかイベントスペースがシャットダウンされるまで回収できません。

enabled については window<%>、border、spacing、alignment については area-container<%>、min-width、min-height、stretchable-width、stretchable-height については area<%> を参照。

Changed in version 1.1 of package `gui-lib`: Added 'fullscreen-button
and 'fullscreen-aux options
for style.
Changed in version 1.66: Allow a dialog% instance
as parent.

## gauge%

> [image: gauge.png]

```

ゲージは有界な整数量の出力値を表示する水平または垂直のバーです。調整可能な範囲があり、現在値は常に 0 からその範囲まで（両端含む）です。値の設定には set-value を使います。

| (is-a?/c panel%) (is-a?/c pane%))                                             |
| style: (listof (or/c 'horizontal 'vertical 'vertical-label 'horizontal-label |
| 'deleted)) = '(horizontal)                                                    |
| (listof (or/c 'horizontal 'vertical                                           |
| 'vertical-label 'horizontal-label                                             |
| 'deleted))                                                                    |
| font: (is-a?/c font%) = normal-control-font                                  |
| enabled: any/c = #t                                                          |
| vert-margin: spacing-integer? = 2                                            |
| horiz-margin: spacing-integer? = 2                                           |
| min-width: (or/c dimension-integer? #f) = #f                                 |

| stretchable-height: any/c = (memq 'vertical style)                           |

| (or/c (is-a?/c frame%) (is-a?/c dialog%)                                      |
|       (is-a?/c panel%) (is-a?/c pane%))                                       |

|                                                                               |

label が文字列ならゲージのラベルとして使われます。そうでなければラベルは表示されません。

label に & が含まれると特別に解析されます。Windows と X では & の次の文字が下線付きになりニーモニックを示します（Mac OS では下線は表示されません）。ゲージにとってニーモニックは意味を持ちませんが、他コントロールとの一貫性のためサポートされています。

font はコントロールのフォントを決めます。enabled については window<%>、horiz-margin と vert-margin については subarea<%>、min-width、min-height、stretchable-width、stretchable-height については area<%> を参照。

+--------------------------------------------------------+
```

```
+-----------------------------------------------+

|                                               |

+-----------------------------------------------+
```
