# 11 合成による設計 (Design by Composition)

This chapter is about the design of programs through composition.

この章は、合成によるプログラムの設計についてです。

## 11.1 The list Function

The list function is a convenient way to create lists with a fixed number of elements.

list 関数は、固定数の要素を持つリストを便利に作成する方法です。

```racket
(list 1 2 3)
```

is equivalent to

は以下と等価です。

```racket
(cons 1 (cons 2 (cons 3 '())))
```

Using list often makes examples much more readable.

list を使うと、例がずっと読みやすくなることが多いです。

## 11.2 Composing Functions

When you have a problem that can be solved by first solving a simpler problem and then combining the results, you are designing by composition.

より単純な問題をまず解き、その結果を組み合わせることで解決できる問題に直面したとき、あなたは合成による設計を行っています。

Consider the problem of sorting a list of numbers.

数値のリストをソートする問題を考えましょう。

One way to sort a list is to repeatedly insert the first item of the unsorted part into the correct position in the already sorted part.

リストをソートする1つの方法は、未ソート部分の最初の項目を、すでにソート済みの部分の正しい位置に繰り返し挿入することです。

First we design a function that inserts a number into a sorted list of numbers.

まず、数値をソート済みの数値リストに挿入する関数を設計します。

```racket
; Number List-of-numbers -> List-of-numbers
; inserts n into the sorted list l
(define (insert n l)
  (cond
    [(empty? l) (list n)]
    [else (if (<= n (first l))
              (cons n l)
              (cons (first l) (insert n (rest l))))]))
```

```racket
(check-expect (insert 5 '()) (list 5))
(check-expect (insert 5 (list 1 3 7)) (list 1 3 5 7))
```

Now we can design sort by composing insert with a recursive call.

次に、insert と再帰呼び出しを合成して sort を設計できます。

```racket
; List-of-numbers -> List-of-numbers
; sorts l in ascending order
(define (sort l)
  (cond
    [(empty? l) '()]
    [else (insert (first l) (sort (rest l)))]))
```

```racket
(check-expect (sort (list 3 2 1)) (list 1 2 3))
```

## 11.3 Auxiliary Functions that Recur

The insert function recurs on the rest of the list.

insert 関数はリストの残りに対して再帰します。

The sort function also recurs on the rest of the list.

sort 関数もリストの残りに対して再帰します。

Both are recursive auxiliary functions.

どちらも再帰的な補助関数です。

## 11.4 Auxiliary Functions that Generalize

Sometimes you discover that a function you wrote for a specific purpose can be made more general.

特定の目的のために書いた関数をより一般的なものにできることに気づくことがあります。

For example, instead of writing a function that only inserts numbers, you might later write a more general insertion function that works for any comparable data.

例えば、数値だけを挿入する関数を書く代わりに、後で任意の比較可能なデータに対して動作するより一般的な挿入関数を書くかもしれません。

Design by composition encourages you to create small, reusable pieces and then assemble them into larger solutions.

合成による設計は、小さく再利用可能な部品を作り、それらを組み合わせてより大きな解決策を構築することを促します。
