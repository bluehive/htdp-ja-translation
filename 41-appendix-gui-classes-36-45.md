## editor-data-class-list<%>

```
+-------------------------------------------------+
| interfaceeditor-data-class-list<%>: interface? |
+-------------------------------------------------+
+-------------------------------------------------+
```

各イベントスペースは `editor-data-class-list<%>` のインスタンスを持ち、`(get-the-editor-data-class-list)` で取得します。新しいインスタンスを直接生成することはできません。このリストはエディタデータクラスの一覧を保持します。この一覧は、ファイルからスニップを読み込むために必要です。Editor Data も参照してください。

```
+--------------------------------------------------------+
| [メソッド]                                             |
|                                                        |
| (send an-editor-data-class-list add snipclass) → void? |
| snipclass: (is-a?/c editor-data-class%)               |
+--------------------------------------------------------+
```

スニップデータクラスをリストに追加します。同じ名前のクラスがすでにリストにある場合、このクラスは追加されません。

```
+--------------------------------------------+
| [メソッド]                                 |
|                                            |
| (send an-editor-data-class-list find name) |
| → (or/c (is-a?/c snip-class%) #f)          |
| name: string?                             |
+--------------------------------------------+
```

与えられた名前を持つスニップデータクラスをリストから探し、見つからなければ `#f` を返します。

```
+------------------------------------------------------+
| [メソッド]                                           |
|                                                      |
| (send an-editor-data-class-list find-position class) |
| → exact-nonnegative-integer?                         |
| class: (is-a?/c editor-data-class%)                 |
+------------------------------------------------------+
```

指定されたクラスのリスト内インデックスを返します。

```
+------------------------------------------+
| [メソッド]                               |
|                                          |
| (send an-editor-data-class-list nth n)   |
| → (or/c (is-a?/c editor-data-class%) #f) |
| n: exact-nonnegative-integer?           |
+------------------------------------------+
```

リストの n 番目のクラス（0 から数える）を返します。リストのクラス数が `n` 以下なら `#f` を返します。

```
+-----------------------------------------+
| [メソッド]                              |
|                                         |
| (send an-editor-data-class-list number) |
| → exact-nonnegative-integer?            |
+-----------------------------------------+
```

リスト内のエディタデータクラスの数を返します。

## editor-data-class%

```
+----------------------------------+
| classeditor-data-class%: class? |
+----------------------------------+
| superclass: object%              |
+----------------------------------+
```

`editor-data-class%` オブジェクトは、`editor-data%` オブジェクトの型を定義します。Editor Data も参照してください。

```
+---------------------------------------------------------+
| [コンストラクタ]                                        |
|                                                         |
| (new editor-data-class%) → (is-a?/c editor-data-class%) |
+---------------------------------------------------------+
```

（役に立たない）インスタンスを生成します。

```
+-----------------------------------------------------+
| [メソッド]                                          |
|                                                     |
| (send an-editor-data-class get-classname) → string? |
+-----------------------------------------------------+
```

クラスの名前を取得します。`wx` で始まる名前は内部用に予約されています。

```
+------------------------------------+
| [メソッド]                         |
|                                    |
| (send an-editor-data-class read f) |
| → (or/c (is-a?/c editor-data%) #f) |
| f: (is-a?/c editor-stream-in%)    |
+------------------------------------+
```

与えられたストリームから新しいデータオブジェクトを読み込み、エラーがあれば `#f` を返します。

```
+-----------------------------------------------------+
| [メソッド]                                          |
|                                                     |
| (send an-editor-data-class set-classname v) → void? |
| v: string?                                         |
+-----------------------------------------------------+
```

クラスの名前を設定します。`wx` で始まる名前は内部用に予約されています。

エディタデータクラス名は通常 `"(lib ...)"` の形にして、クラスのオンデマンド読み込みを可能にするべきです。詳細は Editor Data を参照してください。

## editor-data%

```
+----------------------------+
| classeditor-data%: class? |
+----------------------------+
| superclass: object%        |
+----------------------------+
```

`editor-data%` オブジェクトは、エディタ内のスニップまたは領域に関連付けられた追加データを含みます。Editor Data も参照してください。

```
+---------------------------------------------+
| [コンストラクタ]                            |
|                                             |
| (new editor-data%) → (is-a?/c editor-data%) |
+---------------------------------------------+
```

`get-next` が返す要素は `#f` に初期化されます。

```
+------------------------------------------+
| [メソッド]                               |
|                                          |
| (send an-editor-data get-dataclass)      |
| → (or/c (is-a?/c editor-data-class%) #f) |
+------------------------------------------+
```

このデータのクラスを取得します。

```
+------------------------------------+
| [メソッド]                         |
|                                    |
| (send an-editor-data get-next)     |
| → (or/c (is-a?/c editor-data%) #f) |
+------------------------------------+
```

エディタデータ要素のリストにおける次の要素を取得します。`#f` がリストの終端です。

```
+-----------------------------------------------+
| [メソッド]                                    |
|                                               |
| (send an-editor-data set-dataclass v) → void? |
| v: (is-a?/c editor-data-class%)              |
+-----------------------------------------------+
```

このデータのクラスを設定します。

```
+------------------------------------------+
| [メソッド]                               |
|                                          |
| (send an-editor-data set-next v) → void? |
| v: (or/c (is-a?/c editor-data%) #f)     |
+------------------------------------------+
```

エディタデータ要素のリストにおける次の要素を設定します。`#f` がリストの終端です。

```
+------------------------------------------+
| [メソッド]                               |
|                                          |
| (send an-editor-data write f) → boolean? |
| f: (is-a?/c editor-stream-out%)         |
+------------------------------------------+
```

仕様:
指定されたストリームへデータを書き込み、成功すれば `#t`、そうでなければ `#f` を返します。

既定の実装:
`#f` を返します。

## editor-snip%

```
+----------------------------+
| classeditor-snip%: class? |
+----------------------------+
| superclass: snip%          |
+----------------------------+
```

`editor-snip%` オブジェクトは、`editor<%>` オブジェクトを含み表示する `snip%` オブジェクトです。このスニップクラスは、別のエディタ内にエディタを単一の項目として挿入するために使います。

```
+-------------------------------------------------------------------+
| [コンストラクタ]                                                  |
|                                                                   |
| (new editor-snip%                                                 |
| → (is-a?/c editor-snip%)                                          |
| editor: (or/c (is-a?/c editor<%>) #f) = #f                       |
| with-border?: any/c = #t                                         |
| left-margin: exact-nonnegative-integer? = 5                      |
| top-margin: exact-nonnegative-integer? = 5                       |
| right-margin: exact-nonnegative-integer? = 5                     |
| bottom-margin: exact-nonnegative-integer? = 5                    |
| left-inset: exact-nonnegative-integer? = 1                       |
| top-inset: exact-nonnegative-integer? = 1                        |
| right-inset: exact-nonnegative-integer? = 1                      |
| bottom-inset: exact-nonnegative-integer? = 1                     |
| min-width: (or/c (and/c real? (not/c negative?)) 'none) = 'none  |
| max-width: (or/c (and/c real? (not/c negative?)) 'none) = 'none  |
| min-height: (or/c (and/c real? (not/c negative?)) 'none) = 'none |
| max-height: (or/c (and/c real? (not/c negative?)) 'none) = 'none |
+-------------------------------------------------------------------+
```

`editor` が非 `#f` の場合、スニップが含むエディタとして使われます。`set-editor` も参照してください。

`with-border?` が `#f` でない場合、スニップの周囲に枠線が描画されます。エディタ表示は、`-margin` 引数で指定された量だけスニップ領域内にインセットされます。枠線は `-inset` 引数で指定されたインセットで描画されます。

インセットとマージン引数については `get-inset` および `get-margin` を参照してください。

```
+------------------------------------+
| [メソッド]                         |
|                                    |
| (send an-editor-snip adjust-cursor |
| → (or/c (is-a?/c cursor%) #f)      |
| dc: (is-a?/c dc<%>)               |
| x: real?                          |
| y: real?                          |
| editorx: real?                    |
| editory: real?                    |
| event: (is-a?/c mouse-event%)     |
+------------------------------------+
```

`snip%` の `adjust-cursor` をオーバーライドします。

埋め込みエディタの `adjust-cursor` メソッドを呼んでカーソルを取得します。

```
+--------------------------------------------------+
| [メソッド]                                       |
|                                                  |
| (send an-editor-snip border-visible?) → boolean? |
+--------------------------------------------------+
```

スニップの周囲に枠線が描画されていれば `#t`、そうでなければ `#f` を返します。

`show-border` も参照してください。

```
+-----------------------------------------------------+
| [メソッド]                                          |
|                                                     |
| (send an-editor-snip get-align-top-line) → boolean? |
+-----------------------------------------------------+
```

スニップが align-top-line モードかどうかを報告します。詳細は `get-extent` を参照してください。

`set-align-top-line` も参照してください。

```
+----------------------------------------------------------+
| [メソッド]                                               |
|                                                          |
| (send an-editor-snip get-editor)                         |
| → (or/c (or/c (is-a?/c text%) (is-a?/c pasteboard%)) #f) |
+----------------------------------------------------------+
```

スニップが含むエディタを返します。エディタがなければ `#f` です。

```
+------------------------------------------------------------------+
| [メソッド]                                                       |
|                                                                  |
| (send an-editor-snip get-extent                                  |
| dc: (is-a?/c dc<%>)                                             |
| x: real?                                                        |
| y: real?                                                        |
| w: (or/c (box/c (and/c real? (not/c negative?))) #f) = #f       |
| h: (or/c (box/c (and/c real? (not/c negative?))) #f) = #f       |
| descent: (or/c (box/c (and/c real? (not/c negative?))) #f) = #f |
| space: (or/c (box/c (and/c real? (not/c negative?))) #f) = #f   |
| lspace: (or/c (box/c (and/c real? (not/c negative?))) #f) = #f  |
| rspace: (or/c (box/c (and/c real? (not/c negative?))) #f) = #f  |
+------------------------------------------------------------------+
```

`snip%` の `get-extent` をオーバーライドします。

自身のエディタの `get-extent` メソッドを呼び、さらにエディタスニップのマージンを加算します。

上側のスペースは常に、エディタの先頭行のスペースにスニップの上マージンを加えたものに対応します。通常、descent はエディタの最終行の descent にスニップの下マージンを加えたものに対応します。ただし、スニップが align-top-line モードの場合（`set-align-top-line` を参照）、descent は先頭行の descent に、エディタの残りの行の高さ、スニップの下マージンを加えたものに対応します。

エディタがテキストエディタの場合、見た目が編集に適するよう、通常は `get-extent` が返すエディタの幅から `1` が引かれます。スニップが tight-text-fit モードの場合（`set-tight-text-fit` を参照）、テキストエディタの幅から `2` が引かれ、テキストエディタが点滅キャレット用に予約する 2 ピクセルが除かれます。さらに tight-text-fit モードでは、エディタの高さから行間隔に等しい量が引かれます。既定では tight-text-fit モードは無効です。

```
+-------------------------------------------------+
| [メソッド]                                      |
|                                                 |
| (send an-editor-snip get-inset l t r b) → void? |
| l: (box/c exact-nonnegative-integer?)          |
| t: (box/c exact-nonnegative-integer?)          |
| r: (box/c exact-nonnegative-integer?)          |
| b: (box/c exact-nonnegative-integer?)          |
+-------------------------------------------------+
```

スニップの現在の枠線インセットを取得します。インセットは、スニップの端と枠線のあいだに残す空間の量を設定します。

`l` ボックスには左インセットが格納されます。
`t` ボックスには上インセットが格納されます。
`r` ボックスには右インセットが格納されます。
`b` ボックスには下インセットが格納されます。

```
+--------------------------------------------------+
| [メソッド]                                       |
|                                                  |
| (send an-editor-snip get-margin l t r b) → void? |
| l: (box/c exact-nonnegative-integer?)           |
| t: (box/c exact-nonnegative-integer?)           |
| r: (box/c exact-nonnegative-integer?)           |
| b: (box/c exact-nonnegative-integer?)           |
+--------------------------------------------------+
```

スニップの現在のマージンを取得します。マージンは、エディタ内容の端とスニップの端のあいだに残す空間の量を設定します。

`l` ボックスには左マージンが格納されます。
`t` ボックスには上マージンが格納されます。
`r` ボックスには右マージンが格納されます。
`b` ボックスには下マージンが格納されます。

```
+------------------------------------------------+
| [メソッド]                                     |
|                                                |
| (send an-editor-snip get-max-height)           |
| → (or/c (and/c real? (not/c negative?)) 'none) |
+------------------------------------------------+
```

スニップの最大表示高さを取得します。ゼロまたは `'none` は最大値がないことを示します。

```
+------------------------------------------------+
| [メソッド]                                     |
|                                                |
| (send an-editor-snip get-max-width)            |
| → (or/c (and/c real? (not/c negative?)) 'none) |
+------------------------------------------------+
```

スニップの最大表示幅を取得します。ゼロまたは `'none` は最大値がないことを示します。

```
+------------------------------------------------+
| [メソッド]                                     |
|                                                |
| (send an-editor-snip get-min-height)           |
| → (or/c (and/c real? (not/c negative?)) 'none) |
+------------------------------------------------+
```

スニップの最小表示高さを取得します。ゼロまたは `'none` は最小値がないことを示します。

```
+------------------------------------------------+
| [メソッド]                                     |
|                                                |
| (send an-editor-snip get-min-width)            |
| → (or/c (and/c real? (not/c negative?)) 'none) |
+------------------------------------------------+
```

スニップの最小表示幅を取得します。ゼロまたは `'none` は最小値がないことを示します。

```
+-----------------------------------------------------+
| [メソッド]                                          |
|                                                     |
| (send an-editor-snip get-tight-text-fit) → boolean? |
+-----------------------------------------------------+
```

スニップが tight-text-fit モードかどうかを報告します。詳細は `get-extent` を参照してください。

`set-tight-text-fit` も参照してください。

```
+---------------------------------------------+
| [メソッド]                                  |
|                                             |
| (send an-editor-snip resize w h) → boolean? |
| w: (and/c real? (not/c negative?))         |
| h: (and/c real? (not/c negative?))         |
+---------------------------------------------+
```

`snip%` の `resize` をオーバーライドします。

スニップの最小および最大の幅と高さを、指定値からスニップ枠線空間を引いた値に設定します。`set-min-width`、`set-max-width`、`set-max-height`、`set-min-height` も参照してください。

また、スニップが所有するエディタの最小および最大幅を、`set-max-width` と `set-min-width` 経由で与えられた幅（スニップ枠線空間を引いたもの）に設定します。

```
+---------------------------------------------------------+
| [メソッド]                                              |
|                                                         |
| (send an-editor-snip set-align-top-line tight?) → void? |
| tight?: any/c                                          |
+---------------------------------------------------------+
```

align-top-line モードを有効または無効にします。詳細は `get-extent` を参照してください。

`get-align-top-line` も参照してください。

```
+-----------------------------------------------------------------+
| [メソッド]                                                      |
|                                                                 |
| (send an-editor-snip set-editor editor) → void?                 |
| editor: (or/c (or/c (is-a?/c text%) (is-a?/c pasteboard%)) #f) |
+-----------------------------------------------------------------+
```

スニップが含むエディタを設定し、スニップ内の古いエディタ（あれば）を解放します。新しいエディタがすでにアドミニストレータを持つ場合、新しいエディタはスニップにインストールされません。

`editor-snip%` オブジェクトがエディタに挿入されていないとき、アドミニストレータを持ちません。このあいだ、含まれるエディタにもアドミニストレータを与えません。したがってアドミニストレータのない含まれるエディタは、アドミニストレータを持つ他のディスプレイへ「離反」できます。含まれるエディタが離反し、スニップが後から別のエディタに挿入されたとき、スニップは裏切り者の含まれるエディタを捨て、含まれるエディタを `#f` に設定します。

```
+-------------------------------------------------+
| [メソッド]                                      |
|                                                 |
| (send an-editor-snip set-inset l t r b) → void? |
| l: exact-nonnegative-integer?                  |
| t: exact-nonnegative-integer?                  |
| r: exact-nonnegative-integer?                  |
| b: exact-nonnegative-integer?                  |
+-------------------------------------------------+
```

スニップの現在の枠線インセットを設定します。インセットは、スニップの端と枠線のあいだに残す空間の量を設定します。

```
+--------------------------------------------------+
| [メソッド]                                       |
|                                                  |
| (send an-editor-snip set-margin l t r b) → void? |
| l: exact-nonnegative-integer?                   |
| t: exact-nonnegative-integer?                   |
| r: exact-nonnegative-integer?                   |
| b: exact-nonnegative-integer?                   |
+--------------------------------------------------+
```

スニップの現在のマージンを設定します。マージンは、エディタ内容の端とスニップの端のあいだに残す空間の量を設定します。

```
+--------------------------------------------------+
| [メソッド]                                       |
|                                                  |
| (send an-editor-snip set-max-height h) → void?   |
| h: (or/c (and/c real? (not/c negative?)) 'none) |
+--------------------------------------------------+
```

```
|                                                         |
| (send an-editor-snip use-style-background use?) → void? |
| use?: any/c                                            |
+---------------------------------------------------------+
```

スニップが枠線内の背景描画に、スタイルの背景と透過情報を使う／使わない（既定）ようにします。

`use?` が `#f` の場合、スタイルの背景と透過情報は無視されます。そうでなければ使われます。

## editor-stream-in-base%

```
+--------------------------------------+
| classeditor-stream-in-base%: class? |
+--------------------------------------+
| superclass: object%                  |
+--------------------------------------+
```

`editor-stream-in-base%` オブジェクトは、`editor-stream-in%` オブジェクトがデータの低レベル読み込みを行うために使います。

`editor-stream-in-base%` クラスが直接インスタンス化されることはありませんが、派生クラス `editor-stream-in-bytes-base%` はインスタンス化できます。新しい派生クラスは、この節で述べるすべてのメソッドをオーバーライドしなければなりません。

```
+-------------------------------------------------+
| [メソッド]                                      |
|                                                 |
| (send an-editor-stream-in-base bad?) → boolean? |
+-------------------------------------------------+
```

ストリームからの読み込みでエラーがあった場合は `#t`、そうでなければ `#f` を返します。

```
+-------------------------------------------+
| [メソッド]                                |
|                                           |
| (send an-editor-stream-in-base read data) |
| → exact-nonnegative-integer?              |
| data: (and/c vector? (not immutable?))   |
+-------------------------------------------+
```

`read-bytes` に似ていますが、バイト文字列を埋める代わりに、与えられたベクタを Latin-1 文字で埋めます。このメソッドは既定では `read-bytes` 経由で実装されます。

```
+--------------------------------------------------------+
| [メソッド]                                             |
|                                                        |
| (send an-editor-stream-in-base read-bytes              |
| → exact-nonnegative-integer?                           |
| bstr: (and/c bytes? (not immutable?))                 |
| start: exact-nonnegative-integer? = 0                 |
| end: exact-nonnegative-integer? = (bytes-length bstr) |
+--------------------------------------------------------+
```

与えられたバイト文字列を埋めるためにバイトを読み込みます。戻り値は読み込んだバイト数で、ストリームが空になると要求数より少なくなり得ます。ストリームが空になった場合、次の `bad?` 呼び出しは `#t` を返さなければなりません。

読み込まれたバイトは、`bstr` の位置 `start` から、最大で `end` まで格納されます。

```
+-------------------------------------------------------------+
| [メソッド]                                                  |
|                                                             |
| (send an-editor-stream-in-base read-byte) → (or/c byte? #f) |
+-------------------------------------------------------------+
```

1 バイトを読み込んで返すか、これ以上バイトがなければ `#f` を返します。このメソッドの既定実装は `read-bytes` を使います。

```
+--------------------------------------------------+
| [メソッド]                                       |
|                                                  |
| (send an-editor-stream-in-base seek pos) → void? |
| pos: exact-nonnegative-integer?                 |
+--------------------------------------------------+
```

ストリーム内の指定された絶対位置へ移動します。

```
+------------------------------------------------+
| [メソッド]                                     |
|                                                |
| (send an-editor-stream-in-base skip n) → void? |
| n: exact-nonnegative-integer?                 |
+------------------------------------------------+
```

ストリーム内の次の `n` 文字をスキップします。

```
+--------------------------------------+
| [メソッド]                           |
|                                      |
| (send an-editor-stream-in-base tell) |
| → exact-nonnegative-integer?         |
+--------------------------------------+
```

現在のストリーム位置を返します。

## editor-stream-in%

```
+---------------------------------+
| classeditor-stream-in%: class? |
+---------------------------------+
| superclass: object%             |
+---------------------------------+
```

`editor-stream-in%` オブジェクトは、ファイルまたは他の入力ストリーム（クリップボードなど）からエディタ情報を読み込むために使います。

```
+-----------------------------------------+
| [コンストラクタ]                        |
|                                         |
| (make-object editor-stream-in% base)    |
| → (is-a?/c editor-stream-in%)           |
| base: (is-a?/c editor-stream-in-base%) |
+-----------------------------------------+
```

入力ストリームベース——場合により `editor-stream-in-bytes-base%` オブジェクト——を `base` に与えなければなりません。

```
+----------------------------------------------------------------+
| [メソッド]                                                     |
|                                                                |
| (send an-editor-stream-in get v) → (is-a?/c editor-stream-in%) |
| v: (box/c exact-integer?)                                     |
| (send an-editor-stream-in get v) → (is-a?/c editor-stream-in%) |
| v: (box/c real?)                                              |
+----------------------------------------------------------------+
```

ストリームからデータを読み込み、自身を返します。
不正なストリームからの読み込みは常に `0` を与えます。

`v` ボックスには、ストリーム内の次の整数または浮動小数点値が格納されます。

```
+---------------------------------------------------------------+
| [メソッド]                                                    |
|                                                               |
| (send an-editor-stream-in get-bytes [len]) → (or/c bytes? #f) |
| len: (or/c (box/c exact-nonnegative-integer?) #f) = #f       |
+---------------------------------------------------------------+
```

`get-unterminated-bytes` に似ていますが、最後に読み込んだバイトがヌル終端子であると仮定して破棄します。バイト数が明示されずに `put` の呼び出しで書き込まれたデータにはこのメソッドを使い、バイト数が明示されて書き込まれたデータには `get-unterminated-bytes` を使ってください。

`len` が `#f` でない限り、`len` ボックスにはバイト文字列の長さに 1 を加えた値（終端子を示す）が格納されます。

```
+------------------------------------------------+
| [メソッド]                                     |
|                                                |
| (send an-editor-stream-in get-exact)           |
| → (or/c exact-integer? (and/c real? inexact?)) |
+------------------------------------------------+
```

ストリーム内の次の数値を返します。名前に反して、このメソッドは非正確数を返すことがありますが、それはストリームの次の要素が実際に非正確数として書き込まれた場合に限ります。

```
+--------------------------------------------------+
| [メソッド]                                       |
|                                                  |
| (send an-editor-stream-in get-fixed v)           |
| → (is-a?/c editor-stream-in%)                    |
| v: (box/c (integer-in -9999999999 99999999999)) |
+--------------------------------------------------+
```

`v` ボックスには、`get-fixed-exact` 経由で得られたストリームからの固定サイズ整数が格納されます。

```
+--------------------------------------------+
| [メソッド]                                 |
|                                            |
| (send an-editor-stream-in get-fixed-exact) |
| → (integer-in -9999999999 99999999999)     |
+--------------------------------------------+
```

ストリームから固定サイズ整数を取得します。詳細は `put-fixed` を参照してください。
不正なストリームからの読み込みは常に `0` を与えます。

```
+------------------------------------------------+
| [メソッド]                                     |
|                                                |
| (send an-editor-stream-in get-inexact) → real? |
+------------------------------------------------+
```

ストリーム内の次の浮動小数点値を返します。

```
+---------------------------------------------------------+
| [メソッド]                                              |
|                                                         |
| (send an-editor-stream-in get-unterminated-bytes [len]) |
| → (or/c bytes? #f)                                      |
| len: (or/c (box/c exact-nonnegative-integer?) #f) = #f |
+---------------------------------------------------------+
```

ストリームから次のバイト文字列を返します。これがストリームからバイトを読み戻す推奨の方法です。これに合わせるには、`put` に二つの引数（バイト列の長さを渡す）でデータを書き出してください。

不正なストリームからの読み込みは `#f` または `#""` を返します。

`put` にバイト長が与えられない場合、ヌル終端子用に余分な 1 バイトが含まれることに注意してください。そのようなバイト文字列の読み込みには `get-bytes` を使ってください。

`len` が `#f` でない限り、`len` ボックスにはバイト文字列の長さが格納されます。

```
+------------------------------------------------+
| [メソッド]                                     |
|                                                |
| (send an-editor-stream-in jump-to pos) → void? |
| pos: exact-nonnegative-integer?               |
+------------------------------------------------+
```

ストリーム内の与えられた位置へジャンプします。

```
+-------------------------------------------+
| [メソッド]                                |
|                                           |
| (send an-editor-stream-in ok?) → boolean? |
+-------------------------------------------+
```

ストリームが読み込み可能なら `#t`、そうでなければ `#f` を返します。
不正なストリームからの読み込みは常に `0` または `""` を返します。

```
+----------------------------------------------------+
| [メソッド]                                         |
|                                                    |
| (send an-editor-stream-in remove-boundary) → void? |
+----------------------------------------------------+
```

`set-boundary` を参照してください。

```
+---------------------------------------------------+
| [メソッド]                                        |
|                                                   |
| (send an-editor-stream-in set-boundary n) → void? |
| n: exact-nonnegative-integer?                    |
+---------------------------------------------------+
```

現在のストリーム位置から `n` バイト先にファイル読み込み境界を設定します。この境界を越えて読み込もうとするとエラーが通知されます。境界は `remove-boundary` の呼び出しで取り除かれます。`set-boundary` の各呼び出しは `remove-boundary` の呼び出しと釣り合わなければなりません。

境界は、サブルーチンがデータを読み過ぎて混乱したエラーを起こすのを防ぐのに役立ちます。ただし、悪意あるサブルーチンは自分で `remove-boundary` を呼べます。

```
+-------------------------------------------+
| [メソッド]                                |
|                                           |
| (send an-editor-stream-in skip n) → void? |
| n: exact-nonnegative-integer?            |
+-------------------------------------------+
```

ストリーム内の次の `n` バイトをスキップします。

```
+--------------------------------------------------------------+
| [メソッド]                                                   |
|                                                              |
| (send an-editor-stream-in tell) → exact-nonnegative-integer? |
+--------------------------------------------------------------+
```

現在のストリーム位置を返します。
