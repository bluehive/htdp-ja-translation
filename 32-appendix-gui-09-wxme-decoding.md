# 9 WXME の復号

**原本:** `extracted/appendix/gui/original_markdown_09_WXME_Decoding.md`  
**Source URL path:** `/gui/WXME_Decoding.html`

```
+-----------------+---------------------+
|  (require wxme) | package: `wxme-lib` |
+-----------------+---------------------+
+-----------------+---------------------+
```

`wxme` ライブラリは、`racket/gui` ライブラリ無しで WXME の `editor<%>` 形式ファイル（ファイル形式を参照）を読むための道具を提供します。

```
+---------------------------------+
| [手続き]                        |
|                                 |
| (is-wxme-stream? in) → boolean? |
| in: input-port?                |
+---------------------------------+
```

`in` をのぞき見し、WXME 形式ストリームを示すマジックバイトで始まっていれば `#t` を、そうでなければ `#f` を返します（ファイル形式を参照）。

```
+--------------------------------------------------+
| [手続き]                                         |
|                                                  |
| (wxme-port->text-port in [close?]) → input-port? |
| in: input-port?                                 |
| close?: any/c = #t                              |
+--------------------------------------------------+
```

ストリームが WXME 形式データで始まる入力ポートを受け取り、WXME 内容のテキスト形式を生成する入力ポートを返します。これは DrRacket で WXME ファイルを開き、テキストとして保存した結果に似ています。

`wxme-port->port` と異なり、この関数はスニップに対して、有効なプログラムを無効にしてしまうような自由度の高い扱いをすることがあります。たとえば、wxme ストリーム `in` にビットマップ画像が含まれる場合、妥当なテキストのみの版が存在しないことがあり、その結果 `wxme-port->port` は、もともと妥当な Racket プログラムであったものを構文エラーとなるテキストに変えてしまうかもしれません。それでも、人間が読む用途や、GUI の無い文脈でのみ動くおおまかなプログラム処理ツールには、結果が役立つことがあります。

`close?` が真なら、結果ポートを閉じると元のポートも閉じます。

読めうる非テキスト内容の種類については、「スニップクラスの対応付け」を参照してください。

```
+---------------------------------------------------------+
| [手続き]                                                |
|                                                         |
| (wxme-port->port in [close? snip-filter]) → input-port? |
| in: input-port?                                        |
| close?: any/c = #t                                     |
| snip-filter: (any/c. ->. any/c) = (lambda (x) x)     |
+---------------------------------------------------------+
```

ストリームが WXME 形式データで始まる入力ポートを受け取り、テキスト内容をバイトへ変換し、非テキスト内容を「特殊」値として生成する入力ポートを返します（`read-char-or-special` を参照）。

新しい入力ポートが生成するこれらの特殊値は、ファイルを `editor<%>` オブジェクトに読み込んだときに得られるものとは異なります。`snip%` のインスタンスではなく、特殊値は通常 `object%` の単純な拡張です。読めうる非テキスト内容の種類については、「スニップクラスの対応付け」を参照してください。

`close?` が真なら、結果ポートを閉じると元のポートも閉じます。

`snip-filter` 手続きは、ストリーム用に生成された任意の特殊値に適用され、その結果が代替の特殊値として使われます。

特殊値（フィルタ手続きが生成したものでも可）が `readable<%>` インタフェースを実装するオブジェクトなら、特殊値を得るためにそのオブジェクトの `read-special` メソッドが呼ばれます。

```
+---------------------------+
| [手続き]                  |
|                           |
| (extract-used-classes in) |
| (listof string?)          |
| in: input-port?          |
+---------------------------+
```

二つの値を返します。与えられたストリームが用いるスニップクラス名のリストと、ストリームが用いるデータクラス名のリストです。ストリームが WXME ストリームでなければ、結果は二つの空リストです。与えられたストリームは閉じられず、消費されるのは（もしあれば）WXME ストリーム用のデータだけです。

```
+----------------------------------------------+
| [手続き]                                     |
|                                              |
| (register-lib-mapping! str mod-path) → void? |
| str: string?                                |
| mod-path: (cons/c 'lib (listof string?))    |
+----------------------------------------------+
```

スニップクラス名を、`reader%` 実装を提供する引用付きモジュールパスへ対応付けます。モジュールパスは `'(lib string ...)` の形でなければならず、各 `string` は英数字 ASCII 文字と `.`、`_`、`-`、空白のみを含みます。

```
+--------------------------------------------+
| [手続き]                                   |
|                                            |
| (string->lib-path str gui?)                |
| → (or/c (cons/c 'lib (listof string?)) #f) |
| (or/c (cons/c 'lib (listof string?))       |
| #f)                                        |
| str: string?                              |
| gui?: any/c                               |
|                                            |
| ```racket                                  |
| (or/c (cons/c 'lib (listof string?))       |
|       #f)                                  |
| ```                                        |
+--------------------------------------------+
```

`gui?` が真のときは `editor<%>` モード用、`gui?` が `#f` のときは `wxme` モード用の、`str` に対する引用付きモジュールパスを返します。後者では、組み込みの対応付けと `register-lib-mapping!` で登録された対応付けが使われます。`str` をライブラリパスとして解析できず、対応付けも無い（クラスが組み込みであるか、未知である）場合、結果は `#f` です。

```
+-------------------------------------------------+
| [パラメータ]                                    |
|                                                 |
| (unknown-extensions-skip-enabled) → boolean?    |
| (unknown-extensions-skip-enabled skip?) → void? |
| skip?: any/c                                   |
+-------------------------------------------------+
```

パラメータです。`#f`（既定）に設定されているとき、WXME ストリーム中で認識できないスニップクラスに出会うと例外が発生します。真値に設定されているときは、認識できないスニップクラスのインスタンスは変換後のストリームから単に省略されます。

```
+----------------------------------------+
| [パラメータ]                           |
|                                        |
| (broken-wxme-big-endian?) → boolean?   |
| (broken-wxme-big-endian? big?) → void? |
| big?: any/c                           |
+----------------------------------------+
```

パラメータです。古く短命だった一部の WXME 形式は、ファイルが保存されたマシンのエンディアン順に依存していました。ファイルを読むときに使うエンディアン順を選ぶためにこのパラメータを設定します。既定は現在のプラットフォームのエンディアン順です。

```
+------------------------+
| [手続き]               |
|                        |
| (wxme-read in) → any/c |
| in: input-port?       |
+------------------------+
```

`read` と同様ですが、WXME 形式データで始まるストリーム向けです。WXME データ中に複数の S 式がある場合は、すべて読んで `'begin` でまとめます。

`racket/gui/base` が利用可能（`gui-available?` で判定）なら `open-input-text-editor` が使われます。そうでなければ `wxme-port->port` が使われます。

```
+-------------------------------------------------------------+
| [手続き]                                                    |
|                                                             |
| (wxme-read-syntax source-v in) → (or/c syntax? eof-object?) |
| source-v: any/c                                            |
| in: input-port?                                            |
+-------------------------------------------------------------+
```

`read-syntax` と同様ですが、WXME 形式の入力ストリーム向けです。WXME データ中に複数の S 式がある場合は、すべて読んで `'begin` でまとめます。

`racket/gui/base` が利用可能（`gui-available?` で判定）なら `open-input-text-editor` が使われます。そうでなければ `wxme-port->port` が使われます。

```
+--------------------------------------+
| interface snip-reader<%>: interface? |
+--------------------------------------+
+--------------------------------------+
```

WXME ストリーム中の特定種類のデータ用リーダが実装すべきインタフェースです。インタフェースには二つのメソッド `read-header` と `read-snip` があります。

```
+--------------------------------------+
| [メソッド]                           |
|                                      |
| (send a-snip-reader read-header      |
| version: exact-nonnegative-integer? |
| stream: (is-a?/c stream<%>)         |
| (send a-snip-reader read-snip        |
| (if text-only?                       |
| bytes?                               |
| any/c)                               |
| text-only?: boolean?                |
| version: exact-nonnegative-integer? |
| stream: (is-a?/c stream<%>)         |
|                                      |
| ```racket                            |
| (if text-only?                       |
|     bytes?                           |
|     any/c)                           |
| ```                                  |
+--------------------------------------+
```

`read-header` は、データ型のストリーム固有情報を初期化するために、WXME ストリームあたり高々一度呼ばれます。このメソッドは通常何もしません。

`read-snip` は、ストリーム中でそのデータ型のインスタンスに出会ったときに呼ばれます。このメソッドはデータを読み取り、復号後ストリームの一部として返すバイト列、または復号後ストリームからの「特殊」値として返す任意の種類の値を返します。結果値は、任意で `readable<%>` を実装するオブジェクトでも構いません。

`text-only?` 引数は、`wxme-port->text-port` が呼ばれたときは `#f`、`wxme-port->port` が呼ばれたときは `#t` です。

```
+-----------------------------------+
| interface readable<%>: interface? |
+-----------------------------------+
+-----------------------------------+
```

スニップリーダから返される値が実装すべきインタフェースです。唯一のメソッドは `read-special` です。

```
+-------------------------------------------------+
| [メソッド]                                      |
|                                                 |
| (send a-readable read-special                   |
| source: any/c                                  |
| line: (or/c exact-nonnegative-integer? #f)     |
| column: (or/c exact-nonnegative-integer? #f)   |
| position: (or/c exact-nonnegative-integer? #f) |
+-------------------------------------------------+
```

非グラフィカルモード向けの `read-special` に相当します。値がこのインタフェースを実装しているとき、ソース位置情報を伴ってその `read-special` メソッドが呼ばれ、WXME 復号ポートからの「特殊」結果を得ます。

```
+---------------------------------+
| interface stream<%>: interface? |
+---------------------------------+
+---------------------------------+
```

`snip-reader<%>` インスタンスが用いる WXME 入力ストリームを表します。

```
+--------------------------------------------------------------------------+
| [メソッド]                                                               |
|                                                                          |
| (send a-stream read-integer what)                                        |
| → (or/c (integer-in (- (expt 2 31)) (expt 2 31)) (and/c real? inexact?)) |
| (or/c (integer-in (- (expt 2 31)) (expt 2 31))                           |
| (and/c real? inexact?))                                                  |
| what: any/c                                                             |
| what: any/c                                                             |
| what: any/c                                                             |
| what: any/c                                                             |
| what: any/c                                                             |
| what: any/c                                                             |
|                                                                          |
| ```racket                                                                |
| (or/c (integer-in (- (expt 2 31)) (expt 2 31))                           |
|       (and/c real? inexact?))                                            |
| ```                                                                      |
+--------------------------------------------------------------------------+
```

`get-exact` と同様で、正確な整数または不正確な実数を返します。

`what` フィールドは、ストリームが整数で続かない場合のエラーメッセージ用に、何を読んでいるかを記述します。

ストリーム上には、次の関連メソッドもあります（いずれも `what` の意味は `read-integer` と同じです）。

- `read-fixed-integer` — ストリーム中で固定サイズの正確な整数を読みます（`get-fixed` に類似）。
- `read-inexact` — 不正確な実数を読みます（`get-inexact` に類似）。
- `read-raw-bytes` — 生のバイト列を読みます（`get-unterminated-bytes` に類似）。
- `read-bytes` — 生のバイト列を読みます（`get-bytes` に類似）。
- `read-editor` — 入れ子エディタを読み、エディタ内容を取り出す新しい入力ポートを生成します。

```
+------------------------------------------------+
| [手続き]                                       |
|                                                |
| (read-snip-from-port name who stream) → bytes? |
| name: string?                                 |
| who: any/c                                    |
| stream: (is-a?/c stream<%>)                   |
+------------------------------------------------+
```

`name`（スニップクラスの名前であると期待される）を与え、そのスニップクラスを使って、与えられたストリームの現在位置から読み取ります。`read-snip` メソッドと同様に、処理済みバイト列を返します。

### 9.1 スニップクラスの対応付け

グラフィカルデータが WXME 形式にマーシャルされるとき、ロード時に実装と照合するためのスニップクラス名と関連付けられます。スニップクラスも参照してください。

理想的には、スニップクラス名は次のように生成されます。

```racket
(format "~s" (list '(lib string...)
                   '(lib string...)))
```

ここで、整形されたリストの各要素は引用付きモジュールパスです（`module-path?` を参照）。文字列は英数字 ASCII 文字と `.`、`_`、`-`、空白のみを含まなければならず、`"."` や `".."` であってはなりません。

その場合、第一の引用付きモジュールパスはグラフィカルモードで WXME ファイルをロードするときに使われ、対応するモジュールは `snip-class%` クラスを実装する `snip-class` オブジェクトを提供しなければなりません。第二の引用付きモジュールパスは、グラフィカルサポート無しで WXME ストリームを変換する `wxme` ライブラリが使い、対応するモジュールは `snip-reader<%>` インタフェースを実装する `reader` オブジェクトを提供しなければなりません。当然、`snip-class%` インスタンスと `snip-reader<%>` インスタンスは同じ形式を解析しつつ、文脈（グラフィカルか否か）に適した異なる結果を生成することが期待されます。

スニップクラス名が次のように生成される場合、

```racket
(format "~s" '(lib string...))
```

グラフィカルモードは唯一のモジュールパスを使い、`wxme` には互換対応付けが必要です。`register-lib-mapping!` でインストールしてください。

スニップクラス名が上記どちらの形式でもない場合、グラフィカルモードでは、その名前に対してスニップクラスが登録されているか、組み込みクラス名 `"wxtext"`、`"wxtab"`、`"wximage"`、または（入れ子エディタ用の）`"wxmedia"` のときだけデータを使えます。組み込みクラスでなければ、`wxme` ライブラリには `register-lib-mapping!` でインストールされた互換対応付けが必要です。

いくつかの互換対応付けは `wxme` ライブラリに自動でインストールされます。それらは、各種バージョンの DrRacket がサポートするよく使われるグラフィカル要素——コメントボックス、分数、XML ボックス、Racket ボックス、テキストボックス、`htdp/image` ティーチパック（より一般には `mrlib/cache-image-snip`）が生成する画像、およびテストケースボックス——に対応します。

`wxme-port->port` が作るポートでは、入れ子エディタは `wxme/editor` ライブラリが提供する `editor%` クラスのインスタンスとして表されます。このクラスは、エディタ内容用のポートを返す単一のメソッド `get-content-port` を提供します。画像は `wxme/image` ライブラリが提供する `image%` クラスのインスタンスとして表されます。

コメントボックスは、`editor%` を拡張して `readable<%>` を実装するクラスのインスタンスとして表されます（`wxme/comment` を参照）。read 形式は特殊コメント（`make-special-comment` で作成）を生成するため、`read` でストリームを読むとコメントボックスは消え、特殊コメントの内容は readable インスタンスです。XML、Racket、テキストの各ボックスも同様に、通常どおり展開する `editor%` および `readable<%>` のインスタンスを生成します（`wxme/xml`、`wxme/scheme`、`wxme/text` を参照）。`htdp/image` ティーチパック由来の画像は、`wxme/cache-image` ライブラリの `cache-image%` インスタンスとしてパッケージされます。テストケースボックスは、`wxme/test-case` ライブラリの `test-case%` インスタンスとしてパッケージされます。

#### 9.1.1 入れ子エディタ

```
+------------------------+---------------------+
|  (require wxme/editor) | package: `wxme-lib` |
+------------------------+---------------------+
+------------------------+---------------------+
```

```
+-----------------------+
| class editor%: class? |
+-----------------------+
| スーパークラス: object% |
+-----------------------+
```

テキストモードの WXME ストリーム中の、通常の入れ子エディタのためにインスタンス化されます。

```
+-------------------------------------------------+
| [メソッド]                                      |
|                                                 |
| (send an-editor get-content-port) → input-port? |
+-------------------------------------------------+
```

エディタ内容用のポート（`wxme-port->port` が返すものと同様）を返します。

#### 9.1.2 画像

```
+-----------------------+---------------------+
|  (require wxme/image) | package: `wxme-lib` |
+-----------------------+---------------------+
+-----------------------+---------------------+
```

```
+-------------------------+
| class image%: class?    |
+-------------------------+
| スーパークラス: image-snip% |
+-------------------------+
```

テキストモードの WXME ストリーム中の画像のためにインスタンス化されます。このクラスは `image-snip%` と同様に扱え、同様に振る舞うべきですが、古いコードがなお必要とする場合に備えて、以下のメソッドを追加で持ちます。言い換えると、以下のメソッドは Racket の以前のバージョンとの後方互換のために提供されています。

```
+---------------------------------------------+
| [メソッド]                                  |
|                                             |
| (send an-image get-data) → (or/c bytes? #f) |
+---------------------------------------------+
```

画像用の PNG、XBM、または XPM ファイルのバイト列を返します。

表示幅・表示高さ、および表示画像の左／上に使う実画像内オフセットを返すメソッド `get-w`、`get-h`、`get-dx`、`get-dy` もあります。`get-w`／`get-h` は、データまたはファイル名で指定された実画像の幅／高さと異なることがあり、`-1` は画像データの幅／高さを使うことを意味します。

### 9.2 DrRacket コメントボックス

```
+-------------------------+---------------------+
|  (require wxme/comment) | package: `wxme-lib` |
+-------------------------+---------------------+
+-------------------------+---------------------+
```

```
+-----------------------------------+
| [値]                              |
|                                   |
| reader: (is-a?/c snip-reader<%>) |
+-----------------------------------+
```

コメントボックス用のテキストモードリーダです。

```
+-------------------------------+-------------+
| class comment-editor%: class? |             |
+-------------------------------+-------------+
| スーパークラス: editor%         |             |
| 拡張:                           | readable<%> |
+-------------------------------+-------------+
```

テキストモードの WXME ストリーム中の DrRacket コメントボックスのためにインスタンス化されます。

```
+--------------------------------------------------------------------------------+
| [メソッド]                                                                     |
|                                                                                |
| (send a-comment-editor get-data) → #f                                          |
| method(send a-comment-editor read-special source line column position) → any/c |
| (send a-comment-editor read-special                                            |
| source: any/c                                                                 |
| line: (or/c exact-nonnegative-integer? #f)                                    |
| column: (or/c exact-nonnegative-integer? #f)                                  |
| position: (or/c exact-nonnegative-integer? #f)                                |
+--------------------------------------------------------------------------------+
```

`get-data` は利用可能なデータを返しません（常に `#f`）。

`read-special` は `make-special-comment` を使って特殊コメントを生成します。特殊コメントはコメントテキストを含みます。

### 9.3 DrRacket XML ボックス

```
+---------------------+---------------------+
|  (require wxme/xml) | package: `wxme-lib` |
+---------------------+---------------------+
+---------------------+---------------------+
```

```
+-----------------------------------+
| [値]                              |
|                                   |
| reader: (is-a?/c snip-reader<%>) |
+-----------------------------------+
```

XML ボックス用のテキストモードリーダです。

```
+---------------------------+-------------+
| class xml-editor%: class? |             |
+---------------------------+-------------+
| スーパークラス: editor%     |             |
| 拡張:                       | readable<%> |
+---------------------------+-------------+
```

テキストモードの WXME ストリーム中の DrRacket XML ボックスのためにインスタンス化されます。

```
+----------------------------------------------------------------------------+
| [メソッド]                                                                 |
|                                                                            |
| (send a-xml-editor get-data) → any/c                                       |
| method(send a-xml-editor read-special source line column position) → any/c |
| (send a-xml-editor read-special                                            |
| source: any/c                                                             |
| line: (or/c exact-nonnegative-integer? #f)                                |
| column: (or/c exact-nonnegative-integer? #f)                              |
| position: (or/c exact-nonnegative-integer? #f)                            |
+----------------------------------------------------------------------------+
```

`get-data` は、含まれる XML リテラルから空白が除去されるなら `#t`、そうでなければ `#f` を返します。

`read-special` は、XML を囲む `quasiquote` S 式を生成し、入れ子の Racket ボックス用に `unquote` および `unquote-splicing` による脱出を含めます。

| スーパークラス: editor%        |             |
| 拡張:                         | readable<%> |
+------------------------------+-------------+
```

テキストモードの WXME ストリーム中の DrRacket Racket ボックスのためにインスタンス化されます。

```
+-------------------------------------------------------------------------------+
| [メソッド]                                                                    |
|                                                                               |
| (send a-scheme-editor get-data) → any/c                                       |
| method(send a-scheme-editor read-special source line column position) → any/c |
| (send a-scheme-editor read-special                                            |
| source: any/c                                                                |
| line: (or/c exact-nonnegative-integer? #f)                                   |
| column: (or/c exact-nonnegative-integer? #f)                                 |
| position: (or/c exact-nonnegative-integer? #f)                               |
+-------------------------------------------------------------------------------+
```

`get-data` は、ボックスが splicing な unquote に対応するなら `#t`、非 splicing な unquote なら `#f` を返します。

`read-special` は、ボックス内のコードに対する S 式を生成します。

### 9.8 DrRacket テストケースボックス

```
+---------------------------+---------------------+
|  (require wxme/test-case) | package: `wxme-lib` |
+---------------------------+---------------------+
+---------------------------+---------------------+
```

```
+-----------------------------------+
| [値]                              |
|                                   |
| reader: (is-a?/c snip-reader<%>) |
+-----------------------------------+
```

WXME ストリーム中の DrRacket テストケースボックス用テキストモードリーダです。`test-case%` のインスタンスを生成します。

```
+--------------------------+
| class test-case%: class? |
+--------------------------+
| スーパークラス: object%    |
+--------------------------+
```

テキストモードの WXME ストリーム中の、旧式 DrRacket テストケースボックスのためにインスタンス化されます。

```
+--------------------------------------------------------+
| [メソッド]                                             |
|                                                        |
| (send a-test-case get-comment) → (or/c #f input-port?) |
+--------------------------------------------------------+
```

コメント欄があればそのポートを返します。

関連メソッドには次のものがあります。

- `get-test` — 「test」欄のポートを返します。
- `get-expected` — 「expected」欄のポートを返します。
- `get-should-raise` — 「should raise」欄があればそのポートを返します。
- `get-error-message` — 「error msg」欄があればそのポートを返します。
- `get-enabled?` — テストが有効なら `#t` を返します。
- `get-collapsed?` — テストが折りたたまれていれば `#t` を返します。
- `get-error-box?` — 例外用のテストなら `#t` を返します。
