# 12 プロジェクト: リスト (Projects: Lists)

This chapter presents several extended exercises.

この章では、いくつかの拡張演習を提示します。

All of them aim to solidify your understanding of the elements of design: the design of batch and interactive programs, design by composition, design wish lists, and the design recipe for functions.

それらはすべて、設計の要素についての理解を確かなものにすることを目的としています。バッチプログラムと対話的プログラムの設計、合成による設計、ウィッシュリストの設計、そして関数の設計レシピです。

## 12.1 Real-World Data: Dictionaries

A dictionary can be represented as a list of strings.

辞書は文字列のリストとして表現できます。

```racket
; A Dictionary is a List-of-strings.
```

One useful function counts how many words start with each letter.

有用な関数の1つは、各文字で始まる単語の数を数えるものです。

```racket
; Dictionary -> List-of-Letter-Counts
; counts how often each letter is used as the first letter of a word
(define (count-by-letter dict) ...)
```

Another function finds the most frequent starting letter.

もう1つの関数は、最も頻度の高い先頭文字を見つけます。

```racket
; Dictionary -> Letter-Count
(define (most-frequent dict) ...)
```

## 12.2 Real-World Data: iTunes

An iTunes library can be represented as a list of tracks.

iTunesライブラリはトラックのリストとして表現できます。

```racket
(define-struct track [name artist album time track# added play# played])
; An LTracks is one of:
; - '()
; - (cons Track LTracks)
```

Useful functions include selecting all tracks from a given album and creating sets of unique values.

有用な関数には、指定されたアルバムのすべてのトラックを選択するものや、一意な値の集合を作成するものが含まれます。

```racket
; LTracks String -> LTracks
(define (select-album lt album) ...)
```

## 12.3–12.4 Word Games

Word games often require generating permutations of letters and checking which ones exist in a dictionary.

単語ゲームでは、文字の順列を生成し、辞書に存在するものをチェックすることがよく必要になります。

This is a classic example of design by composition.

これは合成による設計の典型例です。

## 12.5 Feeding Worms

The worm (snake) game can be implemented using a list of positions for the worm's body.

ワーム（スネーク）ゲームは、ワームの体の位置のリストを使って実装できます。

```racket
; A Worm is a List-of-posns.
```

On each clock tick the worm moves, and the list is updated by adding a new head and removing the last segment (unless it just ate food).

各クロックティックでワームは移動し、リストは新しい頭を追加して最後のセグメントを削除することで更新されます（食べた直後を除く）。

## 12.6 Simple Tetris

A simple Tetris game uses a list to represent the landscape of placed blocks.

シンプルなテトリスゲームでは、置かれたブロックの風景をリストで表現します。

```racket
(define-struct tetris [block landscape])
; A Landscape is a List-of-blocks.
```

## 12.7–12.8 Other Games and Finite State Machines

Larger games such as Space War and finite state machines are also naturally expressed using lists.

Space War のようなより大きなゲームや有限状態機械も、リストを使って自然に表現できます。
