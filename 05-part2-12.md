# 12 プロジェクト: リスト (Projects: Lists)

この章では、いくつかの拡張的な練習問題（プロジェクト）を紹介します。これらはすべて、バッチプログラムや対話的プログラムの設計、合成による設計、願望リスト（ウィッシュリスト）の設計、および関数の設計レシピといった、設計の構成要素に対する理解をより確固たるものにすることを目的としています。

最初のセクションでは、現実世界のデータを取り扱います。具体的には、英語の辞書データと iTunes のライブラリ情報です。また、ワードゲーム（単語パズル）に関する課題では、2つのセクションを割いて、関数の合成や問題の本質的な解決に焦点を当てます。残りのセクションでは、ゲームプログラミング（ワームゲーム、テトリス、スペースウォー）や、有限状態機械（FSM）の構築を行います。

## 12.1 現実世界のデータ: 辞書 (Real-World Data: Dictionaries)

現実世界の情報は大量に存在する傾向があり、だからこそプログラムを使ってそれを処理することには大きな意味があります。たとえば、辞書には数個の単語だけでなく、何十万もの単語が含まれています。このような大量の情報を処理するプログラムを設計するときは、小さな例を使用して慎重に進める必要があります。プログラムが正しく動作することを確認できたら、現実世界のデータに対して実行して実際の結果を得ます。もしプログラムの処理速度があまりにも遅い場合は、各関数の動作を振り返り、無駄な計算を排除できないか検討してください。

図74は、英語の辞書全体を読み込むために必要な、わずか1行のコードを示しています。

**図74: 辞書の読み込み**
```racket
; OS X の場合:
(define LOCATION "/usr/share/dict/words")
; LINUX の場合: /usr/share/dict/words または /var/lib/dict/words
; WINDOWS の場合: Linuxを使っている友人からファイルをもらうか、Webから取得してください

; A Dictionary is a List-of-strings.
(define AS-LIST (read-lines LOCATION))
```

図74で示されているコードをあなたのコンピューターに合わせて調整し、`length` 関数を使用して、あなたの辞書にいくつの単語が含まれているかを確認してください（著者らの環境では 2017年7月25日時点で 235,886 単語ありました）。

以下の練習問題では、文字（Letter）が重要な役割を果たします。図74のコードに加えて、プログラムの先頭に以下の定義を追加するとよいでしょう。
```racket
; A Letter is one of the following 1Strings:
; -- "a"
; -- ...
; -- "z"
; または、等価的に次のリストの要素:
(define LETTERS (explode "abcdefghijklmnopqrstuvwxyz"))
```
ヒント：練習問題の例やテストを作成する際には `list` を活用してください。

**練習問題195.** 文字 `Letter` と辞書 `Dictionary` を受け取り、与えられた辞書の中にその文字で始まる単語がいくつあるかをカウントする関数 `starts-with#` を設計してください。関数が正しく動作することを確認したら、あなたのコンピューターの辞書で "e" で始まる単語と "z" で始まる単語がそれぞれいくつあるかを調べてください。

**練習問題196.** 関数 `count-by-letter` を設計してください。この関数は辞書 `Dictionary` を受け取り、その辞書内の各文字が単語の先頭文字として何回使われているかをカウントします。結果は `Letter-Count` のリストになります。`Letter-Count` は、文字とカウント数を組み合わせるデータ型です。まず、このデータ構造（構造体定義など）を設計してから、関数を設計してください。設計が完了したら、あなたのコンピューターの辞書にあるすべての文字について、それぞれの出現回数を調べてください。

**練習問題197.** 関数 `most-frequent` を設計してください。この関数は辞書 `Dictionary` を受け取り、その辞書の中で最も頻繁に単語の先頭文字として出現する文字に対応する `Letter-Count` を返します。あなたのコンピューターの辞書で最も頻繁に使われている先頭文字は何で、それは何回出現しますか？

*設計の選択肢に関する注意*：この練習問題では、前の演習の解決策と、`Letter-Count` のリストから適切なペアを選択する関数を合成することが求められます。後者の関数を設計するには、次の2つのアプローチがあります。
- カウントが最大のペアを選択する関数を設計する。
- ソートされたペアのリストから最初の要素を選択する関数を設計する。
両方の設計を検討してください。どちらが好みですか？その理由は何ですか？

**練習問題198.** 関数 `words-by-first-letter` を設計してください。この関数は辞書 `Dictionary` を受け取り、文字ごとに分類された辞書のリスト（すなわち、辞書のリスト）を返します。
この新しい関数を使用して、練習問題197の `most-frequent` を再設計してください（これを `most-frequent.v2` と呼びます）。設計が完了したら、2つの関数があなたのコンピューターの辞書に対して同じ結果を返すことを確認してください。
```racket
(check-expect (most-frequent AS-LIST) (most-frequent.v2 AS-LIST))
```
*設計の選択肢に関する注意*：`words-by-first-letter` については、与えられた辞書にある特定の文字で始まる単語が1つも存在しない場合の処理に選択肢があります。
- 1つの代替案は、結果のリストから空の辞書を除外することです。これにより、関数のテストや `most-frequent.v2` の設計が簡単になりますが、追加の補助関数を設計する必要があります。
- もう1つの代替案は、該当する単語がない場合でも結果として空リスト `'()` を含めることです。この方法は、除外するための補助関数は不要になりますが、`most-frequent.v2` の設計が少し複雑になります。

## 12.2 現実世界のデータ: iTunes (Real-World Data: iTunes)

Apple の iTunes ソフトウェアは、音楽やビデオなどのコレクション管理に広く使われてきました。あなたの iTunes アプリケーションが収集した情報を解析したいと思うかもしれません。データベースを抽出するのは実は簡単で、メニューから「ライブラリを書き出す」を選択すると、iTunes の情報の XML 表現をエクスポートできます。XML の処理は後のプロジェクトで詳しく扱いますが、ここでは `2htdp/itunes` ティーチパックを使用してデータを取得します。

iTunes ライブラリは、各音楽トラックについて以下のような情報を管理しています。
- トラックID (Track ID): ライブラリ内のユニークな識別子。例: `442`
- 名前 (Name): トラックのタイトル。例: `"Wild Child"`
- アーティスト (Artist): 制作アーティスト。例: `"Enya"`
- album (Album): 所属するアルバムのタイトル。例: `"A Day Without Rain"`
- ジャンル (Genre): 割り当てられた音楽ジャンル。例: `"New Age"`
- 種類 (Kind): ファイルのエンコーディング。例: `"MPEG audio file"`
- サイズ (Size): ファイルサイズ（バイト）。例: `4562044`
- トータルタイム (Total Time): 再生時間（ミリ秒）。例: `227996`
- トラック番号 (Track Number): アルバム内での位置。例: `2`
- トラック数 (Track Count): アルバム内の総トラック数。例: `11`
- 年 (Year): リリース年。例: `2000`
- 追加日 (Date Added): トラックが追加された日時。例: `"2002-7-17 3:55:14"`
- 再生回数 (Play Count): これまでに再生された回数。例: `20`
- 再生日 (Play Date): 最後に再生された日時（UnixタイムスタンプまたはUTC）。

私たちは音楽トラックについて、2つの表現（構造体ベースの表現と、リストベースの表現）を使用します。

図75および図76は、`2htdp/itunes` ティーチパックで実装されている構造体ベースのトラック表現を示しています。

**図75: 構造体によるトラック表現（構造体定義）**
```racket
; ティーチパックのドキュメント（パート 1）:

(define-struct track [name artist album time track# added play# played])
; A Track is a structure:
;   (make-track String String String N N Date N Date)
; 解釈: トラックのタイトル、アーティスト、アルバム、再生時間（ミリ秒）、
; アルバム内でのトラック番号、追加日、再生回数、最終再生日を記録する。

(define-struct date [year month day hour minute second])
; A Date is a structure:
;   (make-date N N N N N N)
; 解釈: 年、月（1〜12）、日（1〜31）、時間（0〜23）、分（0〜59）、秒（0〜59）を記録する。
```

**図76: 構造体によるトラック表現（関連する関数）**
```racket
; create-track : Any Any Any Any Any Any Any Any -> Track or #false
; 引数が正当な値であれば Track 構造体のインスタンスを生成し、そうでなければ #false を返す。

```

これらを使用して、iTunes コレクションに関する情報を計算できます。

**練習問題199.** `Date`、`Track`、および `LTracks` の例を作成してください。これらは以降の練習問題でのテスト入力として役立ちます。

## 12.3 ワードゲーム: 合成 (Word Games: Composition)

多くの言葉遊び（アナグラムやスペリングゲームなど）では、特定の文字の組み合わせから生成できる単語を見つける必要があります。
このセクションでは、与えられた単語の文字を並べ替えて、辞書に存在する意味のある単語（アナグラム）をすべて見つけるプログラムを設計します。

この問題は、以下の2つの主要なタスクに分解できます。
1. 与えられた単語の文字のすべての順列（並べ替え）を生成する。
2. 生成された文字列のリストのうち、辞書に存在する単語だけをフィルタリングする。

**練習問題215.** 単語（文字列）と辞書 `Dictionary` を受け取り、その単語の文字を並べ替えて作ることができる、辞書に存在するすべての単語のリストを返す関数 `alternative-words` を設計してください。

**練習問題216.** 文字列を文字（1String）のリストに変換する `explode` や、文字のリストを文字列に結合する `implode` を使って、単語ゲームプログラムのテストを整理してください。

## 12.4 ワードゲーム: 問題の核心 (Word Games: The Heart of the Problem)

文字のすべての順列を生成することは、このプログラムの最も難しい部分であり、合成による設計の優れた練習になります。

文字のリストを受け取り、そのすべての順列（リストのリスト）を返す関数 `arrangements` を設計します。
```racket
; List-of-1Strings -> List-of-List-of-1Strings
; 与えられた文字リストのすべての順列を生成する
(define (arrangements w)
  '())
```

この再帰関数を設計するために、次のように考えます。
リストの最初の文字 `char` を取り出し、残りの文字リストのすべての順列を再帰的に生成します。そして、その再帰結果の各順列の「すべての可能な位置」に `char` を挿入します。

**練習問題217.** 文字 `1String` と、順列のリスト `List-of-List-of-1Strings` を受け取り、その文字を各順列のすべての可能な挿入位置（先頭、中間、末尾など）に挿入して得られる新しい順列のリストを返す関数 `insert-everywhere/in-all-words` を設計してください。

## 12.6 シンプル・テトリス (Simple Tetris)

テトリスは有名な落ち物パズルゲームです。このゲームの状態をリストと構造体を使って設計します。
```racket
(define-struct tetris [block landscape])
; A Tetris is a structure:
;   (make-tetris Block Landscape)
; A Landscape is a List-of-blocks.
```
ここで、`Block` は現在落下中のブロックの座標を表し、`Landscape` はこれまでに着地して積み上がったブロックのリストです。

**練習問題220.** 与えられた `Tetris` 状態をキャンバスに描画する関数 `tetris-render` を設計してください。

**練習問題221.** ブロックがまっすぐ落下し、床または積み上がった他のブロックに着地すると、次の新しいブロックが生成されて落下を開始する、基本的なテトリスゲーム `tetris-main` を設計してください。

**練習問題222.** プレイヤーが左右の矢印キーを使用して、落下中のブロックを左右に移動できるようにゲームを拡張してください。ブロックが壁を突き抜けたり、すでに積み上がっているブロックと重なったりしないように制御してください。

**練習問題223.** ブロックがキャンバスの最上部に達した（積み上がりすぎた）ときにゲームが終了する機能を追加してください。

## 12.8 有限状態機械 (Finite State Machines)

有限状態機械（FSM）を解釈・シミュレートする一般的なプログラムを設計します。

図82は、FSMのデータ表現を示しています。

**図82: 有限状態機械の表現**
```racket
; An FSM is one of:
;  -- '()
;  -- (cons Transition FSM)

(define-struct transition [current next])
; A Transition is a structure:
;   (make-transition FSM-State FSM-State)

; A FSM-State is a Color.
```
FSMは遷移のリストとして表され、状態は色の名前（文字列）です。任意のキー入力によって状態が遷移します。

**練習問題226.** 2つの状態が等しいかどうかを判定する述語 `state=?` を設計してください。

**練習問題227.** 黒と白が交互に入れ替わる「BWマシン」のFSMデータ表現を作成してください。

**練習問題228.** 現在の `FSM` 状態とキー入力 `KeyEvent` を受け取り、次の状態を遷移テーブルから検索して返す関数 `find-next-state` を設計してください。
